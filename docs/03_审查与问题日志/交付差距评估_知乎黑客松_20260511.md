## 最大缺口

### P0-1：线上 demo 链接缺失

官方要求是“提交可访问的线上 demo，确保评委可实际上手体验”。当前 zip 需要评委手动下载、解压、打开扩展管理页、开启开发者模式、加载插件。这个流程对普通评委不够稳，也不等于“线上 demo 链接”。

建议最短路径：

1. 做一个 `web-demo/` 静态网页版本。
2. 使用现有演示数据 JSON。
3. 不接真实 Chrome API，不接真实知乎页面读取。
4. 页面直接展示：
   - 欢迎 / 产品定位
   - 资产库
   - 知识宇宙
   - 知识球预览
   - 时间轴
   - 总复盘
5. 部署到 Vercel / Netlify / GitHub Pages。
6. 线上 demo 文案说明：“这是演示模式；完整能力在浏览器扩展中读取用户当前打开的知乎文章。”

### P0-2：正式产品说明缺失

当前 README 偏开发安装，旧 product_doc 偏 1.0 项目地图。参赛需要一份正式说明，建议文件名：

`docs/00_总览索引/产品说明_知识图鉴_知乎黑客松.md`

必须覆盖：

- 核心思路：给知乎创作者的长期作品资产系统
- 用户痛点：创作者写了很多，但难以复盘、归档、复用、形成长期主题
- 产品流程：打开知乎文章 → 分析当前文章 → 入资产库 → 点亮知识宇宙 → 周复盘指导下一篇
- 技术方案：Chrome 扩展 / IndexedDB / 本地优先 / AI 分析 / D3 or Three.js 可视化 / 演示数据
- 与知乎生态契合：
  - 帮助知乎创作者复盘自己的公开文章
  - 提升创作者长期留存和持续创作动力
  - 让知乎文章从一次发布变成长期知识资产
  - 未来可接知乎搜索、热榜、直答 Agent、关注流，形成选题与复盘闭环
- 合规说明：
  - 不批量抓取
  - 不绕过权限
  - 只读取用户当前已经打开的页面内容或用户手动输入内容
  - API Key 由用户自己填写并保存在本机

## 次级缺口

### P1：知乎官方 API 接入还没有落地

当前作品仍然主要依赖外部 LLM provider。黑客松提供了直答 Agent、知乎搜索、热榜、故事、关注流等接口。作品不一定必须全部接入，但如果完全不提，容易显得和比赛接口关系弱。

建议做“最小比赛特化”：

1. 产品说明里写清楚未来接入点。
2. 若时间允许，优先接一个“知乎搜索 / 直答 Agent”入口：
   - 在分析结果页新增“用知乎站内讨论补充选题依据”
   - 或在总复盘里新增“下周选题灵感来自知乎热榜/搜索”
3. 如果来不及接真实 API，就做 stub 展示 + 文档说明“比赛 API 权限开通后替换 provider”。

### P1：演示视频 / 路演稿还缺正式提交版

已有 `2.0_demo_script.md`，但它还是插件录制脚本。建议再生成：

- `docs/00_总览索引/3分钟路演稿_知识图鉴.md`
- `docs/00_总览索引/提交页项目详情文案.md`

### P2：GitHub 仓库链接

这是加分项，不是必交。当前项目文档和完成报告很多，代码根目录已经整理过，但还需要：

- 清理不适合上传的临时目录和本地数据
- 确认不含 API Key
- 增加 `.gitignore`
- 写正式 README
- 推 GitHub
