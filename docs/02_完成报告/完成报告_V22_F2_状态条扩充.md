# 完成报告：V22 F2 状态条扩充

时间：2026-05-09

## 本次改动

- `panel.html`
  - 顶部状态条从 2 列改为 5 列：`总 / 年 / 月 / 周 / 日`。
  - 每列统一显示为：`N篇 · N字`。
  - 本周进度条保留在底部整行，文案改为 `本周 N/5 篇`。
  - CSS 改成 5 列紧凑网格，进度条横跨整行。

- `panel.js`
  - 新增 `summarizeTopMetricBuckets(records, now)`，按本地时间切分总/年/月/周/日。
  - 新增 `localPeriodStarts()`、`recordsSince()`、`formatMetricSummary()` 等小函数，避免把日期逻辑塞进 DOM 更新里。
  - `updateTopMetrics()` 改为从 `dbGetAllArticles()` 一次取数，统一计算作品数和正文长度。

- `tests/panel_assets.test.js`
  - 新增纯函数测试：验证总/年/月/周/日的作品数与字数切片。
  - 新增 DOM 测试：验证五列状态条和本周进度条真实渲染。

## 验收标准

- 顶部状态条有 5 列：总、年、月、周、日。
- 每列显示作品数和字数。
- 年/月/周/日按 `savedAt` 本地时间切片。
- 进度条仍在状态条底部整行显示。
- 不改 IndexedDB schema，不改 Markdown/JSON 格式，不碰宇宙视觉参数。

## 实际结果

- 已符合：状态条已扩成 5 列。
- 已符合：总/年/月/周/日都按文章正文 `article.body.length` 累加字数。
- 已符合：本周进度条显示为 `本周 N/5 篇`；F3 会继续把 5 改成用户可设置目标。

## 验证记录

- 前端测试：`100 passed / 0 failed`
- 新增测试：
  - `summarizeTopMetricBuckets: 按总/年/月/周/日统计作品和字数`
  - `updateTopMetrics: 渲染五列状态条和本周进度`

## 风险与回滚

- 风险：窄屏下 5 列状态条信息密度更高，后续 F3/F9 做视觉时要继续留意文字是否拥挤。
- 回滚路径：恢复 `panel.html` 中原 2 列 `metricArticles/metricChars` DOM，并恢复旧版 `updateTopMetrics()`。
