# B1 完成报告 · 知识球图谱模式

## 做了什么

### panel.js
- 新增地图模式状态：
  - `mapMode = 'free'`
  - `mapSimulation`
- 新增 `dominantNodeRole(roleCounts)`：根据当前项目内节点的 `primary / secondary / background` 出现情况，判断节点在知识球里的层级。
- 扩展 `prepareMapData()`：
  - 保留原有自由图谱节点和连线数据。
  - 额外统计每个节点在当前项目里的角色分布。
  - 给每个节点附加 `role` 和 `roleCounts`。
- 新增 `renderMapView(nodes, links, container)`：
  - 在地图 Tab 顶部渲染「自由图谱 / 知识球」切换。
  - 切换模式时停止旧的 D3 force simulation，避免隐藏图谱继续 tick。
- 保留原 `renderMap()` 作为自由图谱模式。
- 新增 `renderKnowledgeSphere()`：
  - 用同心圆显示节点层级。
  - 主节点在内圈，次节点在中圈，背景节点在外圈。
  - 保留节点类型颜色、节点大小、节点关系连线。
  - 点击知识球里的节点，同样进入节点反链详情页。
- `loadMap()` 改为调用 `renderMapView()`，由当前模式决定实际渲染方式。

### panel.html
- 新增地图模式切换工具栏样式：
  - `.map-toolbar`
  - `.map-mode-group`
  - `.map-mode-btn`
  - `.map-mode-hint`
- 保持原有地图 SVG、图例、D3 交互样式不变。

### 问题日志_Claude审核清单.md
- 将 B1「知识球图谱模式」状态从「待做」更新为「已完成」。

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 打开「地图」Tab | 顶部出现「自由图谱 / 知识球」切换按钮 |
| 默认进入地图 | 仍显示原来的自由力导图 |
| 点击「知识球」 | 地图切换为同心圆层级布局 |
| 查看知识球 | 主节点在内圈、次节点在中圈、背景节点在外圈 |
| 查看节点颜色 | 节点仍按 person / concept / theory / event 使用原有颜色 |
| 点击知识球节点 | 进入该节点的反链详情页 |
| 再点击「自由图谱」 | 返回原来的自由力导图 |
| 反复切换模式 | 不出现旧图谱残留或持续抖动 |

## 与预期相比

- 达成 B1 最小可用目标：同一个地图 Tab 内现在有「自由图谱」和「知识球」两种视图。
- 保守实现：没有新增 DB schema，也没有引入节点层级字段；知识球层级直接复用 AI 已返回的 `nodes_hit.role`。
- 与 Obsidian 的差异化方向一致：不是做通用图谱，而是围绕创作者作品中的主次知识节点做专题星系。
- 暂未实现：选择单个专题/知识球的独立入口、项目内多个知识球、节点拖拽固定位置，这些可以放到后续 B1.1 或 B3 视图机制里。
