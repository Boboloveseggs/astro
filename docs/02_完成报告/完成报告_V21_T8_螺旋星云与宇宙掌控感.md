# 完成报告：V21 T8 螺旋星云与宇宙掌控感

## 本次目标

把归档页宇宙从“随机散点 + 中心光圈”的粗糙状态，改成更有规律的螺旋星云；同时解决拉近后的像素感，并让宇宙像知识星球一样可以被拖拽旋转、滚轮缩放。

## 实际改动

### `panel.js`

- 增加宇宙缩放常量：`UNIVERSE_ZOOM_MIN / MAX / STEP` 与拖拽旋转灵敏度。
- 将星点贴图升级为更高分辨率：
  - 普通星点贴图：`160 -> 256`
  - 核心柔光贴图：`256 -> 384`
  - 开启 `LinearMipmapLinearFilter`，降低拉近时的像素格感。
- 将程序星云底图升级为 `2400 x 1500`，并记录 `texture.userData.width / height` 供测试验证。
- 新增 `createUniverseSpiralArmCloud()`：
  - 使用 2 条主螺旋臂。
  - 生成 `22000` 个规则螺旋粒子。
  - 粒子由紫黑金冷暖色混合构成，不再只靠一张贴图表现星云。
- 调整知识星球坐标 `universe3DPoint()`：
  - 从偏随机散布改为沿螺旋臂落位。
  - 降低 z 方向抖动，让全景更像一个可识别的星系盘。
- 调整中心光源与背景星云：
  - 降低中心核心光晕尺寸和透明度，避免“中间一坨光圈”压住螺旋结构。
  - 将 `galaxyPlane` 透明度降到 `0.44`，让真实粒子层成为主视觉。
- 降低跨星球连线全景透明度：`0.018 -> 0.004`，避免全景出现蛛网感。
- 增加宇宙拖拽旋转：
  - `pointerdown / pointermove / pointerup`
  - 横向拖拽旋转星云盘，纵向拖拽调整倾斜角。
  - 拖拽时暂停 hover 与点击误触。
- 增加宇宙滚轮缩放：
  - 滚轮向上拉近，向下拉远。
  - 缩放范围：`0.48 -> 1.85`。
  - 缩放时关闭正在进行的镜头 tween，避免用户操作和自动镜头互相抢。
- 点击知识星球后自动把目标缩放推进到 `0.74`，形成更强的“镜头进入星云”感。
- 保留慢速自转，并在用户拖拽/缩放后的数秒内降低自转速度，避免抢手感。

### `panel.html`

- 将宇宙 canvas 鼠标样式改为 `grab / grabbing`。
- 增加 `touch-action: none` 与 `user-select: none`，避免拖拽时选中文本或出现蓝色选中颗粒感。

### `tests/universe_fixture.html`

- 同步宇宙 canvas 的拖拽鼠标样式，保证截图 fixture 与真实面板一致。

### `tests/panel_map.test.js`

- 在现有宇宙渲染测试中追加验证：
  - 星云底图宽度至少 `2000`，防止低清贴图回退。
  - `spiralCloud` 存在，且粒子数至少 `9000`。
  - `spiralCloud.userData.mode === 'spiral'`。
  - 滚轮向上能让 `targetZoom` 变小，也就是拉近镜头。
  - 拖拽能改变 `userRotation.z`，证明宇宙可被手动旋转。

## 风险与回滚

- 风险：粒子数从原来的几千级提升到 `22000` 螺旋粒子，低端机器可能比之前更吃 GPU。
- 缓解：仍然只在地图宇宙页初始化，离开地图页会 `stopUniverseScene()` 清理 renderer。
- 回滚路径：
  1. 删除 `createUniverseSpiralArmCloud()`。
  2. 移除 `spiralCloud` 创建、添加和测试断言。
  3. 将 `universeTexture()` 尺寸、`universeGalaxyTexture()` 尺寸、`galaxyPlane` 透明度和中心光晕参数恢复到上一版。
  4. 删除 `setUniverseTargetZoom()` 与 `bindUniversePointerEvents()` 中的拖拽/滚轮逻辑。
