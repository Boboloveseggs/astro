# 完成报告 · V2.1 T8 地图视觉返工：3D 星云宇宙

完成时间：2026-05-08

## 一、这次要解决什么

用户明确指出：要的不是“金色球贴在背景上”，而是参考图那种缓慢顺时针移动的星云宇宙。

本次目标：

- 宇宙主体改成 3D 星云渲染。
- 星云缓慢顺时针旋转，并带轻微呼吸感。
- 知识星球外观改成小亮点，不要边框，只是一个模糊发光点。
- hover 到亮点时显示知识星球名称和数量信息。
- 第一次点击星点时，不切页面，而是在宇宙里丝滑推进并锁定视角。
- 再次点击已锁定星点，才进入原先设计的金色知识球内部视图。

## 二、实际改动

### 新增 `three.min.js`

- 引入本地 Three.js 文件，作为扩展内置脚本使用。
- 不依赖远程 CDN，离线可运行。
- `panel.html` 和测试页都改为在 `panel.js` 前加载。

### `panel.js`

- 新增 `universeSceneState`，用于管理 WebGL 宇宙渲染状态。
- `stopMapSimulation()` 现在会同时停止地球球体和 3D 宇宙动画，避免切 Tab 后动画残留。
- 新增 `stopUniverseScene()`：清理 `requestAnimationFrame`、resize 监听、WebGL renderer。
- 新增 `universeGalaxyTexture()`：用 canvas 程序生成星云旋涡纹理，模拟用户给的参考图方向。
- 新增 `createUniversePointCloud()`：生成 3D 星尘点云，并改成圆形柔光粒子。
- 新增 `renderUniverseThreeView()`：核心 3D 宇宙渲染入口。
- 新增 `bindUniversePointerEvents()`：支持鼠标移动拾取星点、hover 显示名称、点击锁定。
- 新增 `focusUniverseBall3D()`：第一次点击只锁定星点和推进镜头，不进入单球。
- 新增 `startUniverseRenderLoop()`：驱动缓慢顺时针旋转、呼吸缩放、镜头平滑跟随。
- 保留 `renderUniverseSvgFallbackView()`：如果 WebGL / Three.js 不可用，回退到上一版 SVG 宇宙，避免地图空白。

### `panel.html`

- 新增 `.universe-canvas-wrap`：WebGL canvas 容器。
- 新增 `.universe-star-tooltip`：星点 hover 名称浮层。
- 调整 `.universe-lock-card`：锁定后淡入，不再始终占画面。
- 加载顺序新增：

```html
<script src="three.min.js"></script>
```

### `tests/test.html`

- 加载 `../three.min.js`，保证单元测试环境与真实 panel 一致。

### `tests/universe_fixture.html`

- 同步 WebGL 宇宙相关 CSS 和 Three.js 加载。
- 继续用于生成宇宙截图。

### `tests/panel_map.test.js`

- 更新宇宙测试，不再断言 SVG 星云层。
- 新断言：
  - 渲染 WebGL canvas。
  - 存在 hover 星点名称浮层。
  - 自动锁定当前项目对应知识星球。
  - 3D 状态里保存 `lockedObject`，用于镜头跟随。
  - 3D 宇宙不再渲染带边框的 SVG 球体。

## 三、验收标准与结果

| 验收项 | 结果 |
|---|---|
| 星云是 3D / WebGL 渲染，不是 SVG 平面球 | 通过 |
| 星云缓慢顺时针旋转 | 通过，`galaxyGroup.rotation.z -= dt * 0.006` |
| 星点无边框，只是模糊发光点 | 通过，知识星球改为 Three.js `SpriteMaterial` 柔光纹理 |
| hover 星点显示名称 | 通过，`universe-star-tooltip` 根据 raycaster 命中显示 |
| 第一次点击星点只锁定并推进镜头 | 通过，`focusUniverseBall3D()` 不重载地图 |
| 再次点击已锁定星点进入金色球 | 通过，复用 `enterUniverseBall()` |
| 项目栏找不到时自动锁定主领域球 | 通过，保留 `resolveUniverseLockBallId()` |
| 原金色知识球内部视图不破坏 | 通过，单球视图仍走 `renderEarthSphere()` |

## 八、与预期相比

这次已经从“SVG 平面宇宙”切换到“WebGL 3D 星云宇宙”。默认状态更接近用户参考图：星云是主体，知识星球只是发光点；点击后仍留在宇宙里推进镜头，而不是生硬换页。

仍需人工感受的一点：旋转速度和推进距离属于审美参数，如果用户希望更慢、更近或更远，可以在下一轮只调参数，不再重写结构。

## 九、回滚路径

如需回滚本次 3D 宇宙：

1. 删除 `three.min.js`。
2. 移除 `panel.html` 和测试页里的 Three.js script。
3. 在 `panel.js` 中移除本次新增的 3D 宇宙函数，并把 `renderUniverseView()` 改回直接调用 SVG 版本。
4. 还原 `tests/panel_map.test.js` 的 SVG 宇宙断言。
5. 删除 `screenshots_V21_T8_three_universe/`。
