# 完成报告：P0 项目隔离自我审查补强

日期：2026-05-04

## 做了什么

- 对刚完成的 P0「同 URL 项目隔离」改动做自我审查。
- 发现并修复两个边界风险：
  - 备份导入时，同项目同 URL 覆盖会让文章使用现有 id，但备份里的 `nodes.articles` 仍可能引用旧 id。
  - 本地反向同步的 shortId 如果发生末 6 位碰撞，可能误更新某一条记录。
- `dbImportAll()` 现在会建立备份旧 articleId 到实际保存 articleId 的映射。
- `dbImportAll()` 导入每篇文章后，会用 `dbReplaceArticleNodes()` 重建该文章节点引用。
- 新增 `findUniqueShortIdRecord()`，shortId 只有唯一命中时才允许反向同步兜底。
- 补充 shortId 碰撞测试。

## 验收标准

- 备份导入同项目同 URL 覆盖时，节点引用应指向实际保存后的文章 id。
- 导入覆盖旧文章时，旧节点引用应被清理并按新 `nodes_hit` 重建。
- shortId 末 6 位碰撞时，本地文件夹反向同步不能随机更新任意一条记录。
- 测试页通过，且测试数量更新。

## 与预期相比

- 这次审查发现的是低概率但和 P0 数据安全相关的边界问题，已在本轮直接修复。
- 没有新增 schema，也没有改变 Markdown/JSON 文件格式。
- 仍保留「同项目内 URL 唯一，不同项目同 URL 可共存」的产品规则。
