# 完成报告 · V2.1 T8 沉浸式镜头与单球缩放修正

完成时间：2026-05-09

## 一、这次要解决什么

用户反馈了 3 个问题：

- 点击知识星点后，希望不是继续俯瞰放大，而是像进入宇宙内部，产生身临其境的视角转换。
- 默认宇宙一开始太大；进入金色知识球后也太大，需要手动拖/缩放才看得到完整结构。
- 金色知识球旋转时，文字会变蓝，疑似被浏览器选中；同时希望颗粒感更少、更顺滑。

## 二、实际改动

### `panel.js`

- 新增 `EARTH_SPHERE_INITIAL_ZOOM = 0.84`。
- `renderEarthSphere()` 初始 zoom 从 `1` 改为 `0.84`，进入金色知识球时默认能看见更完整的球体结构。
- `resetEarthSphereView()` 也回到 `0.84`，避免“回到中心”后又变得过大。
- `startEarthSphereDrag()` 增加 `event.preventDefault()`，拖拽球体时阻止浏览器文本选中。
- `universeCameraTargetForObject()` 改为沉浸式斜向视角：
  - 全局宇宙镜头从 `z=430` 拉远到 `z=620`，默认宇宙更收。
  - 锁定星点时不再正俯视放大，而是沿星云平面做斜向飞入。
- `startUniverseRenderLoop()` 的入场镜头从 `z=760` 开始，形成“先看到小宇宙，再进入”的节奏。

### `panel.html`

- `#earthSphere` 增加 `user-select: none` / `-webkit-user-select: none`。
- `.sphere-node` 和子元素禁用文本选择。
- `.sphere-node-core` 增加 `shape-rendering: geometricPrecision`。
- `.sphere-node-label` 增加 `text-rendering: geometricPrecision`，并禁用文本选择。

### `tests/universe_fixture.html`

- 同步金色知识球的抗选中与顺滑渲染 CSS。

### `tests/earth_sphere_fixture.html`

- 同步抗选中与顺滑渲染 CSS，保证截图 fixture 与真实面板一致。

### `tests/panel_map.test.js`

- 增加断言：金色知识球初始 zoom 应等于 `EARTH_SPHERE_INITIAL_ZOOM`。
- 增加断言：锁定星点后的镜头应贴近星云平面，并具备斜向飞入位移，而不是正俯视放大。

## 八、与预期相比

这次主要修“镜头”和“操作质感”：

- 默认宇宙从大画面俯瞰改为更远的收纳视角。
- 点击星点后的视角从俯瞰推进改为斜向贴近星云的沉浸式飞入。
- 金色知识球默认更小，减少刚进入时需要手动调整的问题。
- 拖拽旋转时禁止文字选择，解决文字变蓝的根因。

## 九、回滚路径

如需回滚本次改动：

1. 删除 `EARTH_SPHERE_INITIAL_ZOOM`，把单球 zoom 恢复为 `1`。
2. 还原 `universeCameraTargetForObject()` 和 `startUniverseRenderLoop()` 的镜头参数。
3. 移除 `startEarthSphereDrag()` 中的 `event.preventDefault()`。
4. 移除 `panel.html`、`tests/universe_fixture.html`、`tests/earth_sphere_fixture.html` 中新增的抗选中与渲染样式。
5. 还原 `tests/panel_map.test.js` 中本次新增断言。
6. 删除 `screenshots_V21_T8_immersive_camera/`。
