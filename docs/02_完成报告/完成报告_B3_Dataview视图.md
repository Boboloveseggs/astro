# B3 完成报告 · Dataview 视图

## 做了什么

### panel.js
- 新增资产视图状态：
  - `assetViewFilters`
  - `savedAssetViews`
- `initAssetsToolbar()` 扩展为资产工具台：
  - 保留原搜索框。
  - 新增保存视图下拉框。
  - 新增「筛选」按钮。
  - 新增筛选面板。
- 筛选面板支持 4 类条件：
  - 年份：按 `savedAt` 前缀过滤，例如 `2024`
  - 领域：匹配 `analysis.domain` / `analysis.sub_domain`
  - 节点：匹配 `analysis.nodes_hit[].name`
  - 类型：全部、已分析作品、速记、专栏、回答、想法、手动
- 新增保存视图能力：
  - 使用 `dbSetConfig('asset_views', savedAssetViews)` 保存到 IndexedDB config。
  - 不改变文章数据结构。
  - 同名视图会被新配置覆盖。
- 新增清除条件能力：
  - 点击「清除条件」恢复全部资产。
  - 保存视图下拉框选回「视图」时也会清空当前条件。
- `loadAssets()` 现在支持搜索词和视图条件叠加过滤。
- 空结果文案会显示当前视图摘要，方便判断是搜索无结果还是视图条件无结果。

### panel.html

  - `.asset-view-input`
  - `.asset-view-actions`

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 打开「资产」Tab | 顶部仍有搜索框，并出现「视图」下拉框和「筛选」按钮 |
| 点击「筛选」 | 展开年份、领域、节点、类型条件面板 |
| 输入年份 `2024` | 只显示 `savedAt` 属于 2024 年的资产 |
| 输入领域 `哲学` | 只显示领域或子领域包含「哲学」的资产 |
| 输入节点名 | 只显示 `nodes_hit` 中包含该节点的资产 |
| 类型选择「速记」 | 只显示 B2 创建的速记资产 |
| 同时输入年份、领域、节点 | 多个条件叠加生效 |
| 再输入搜索关键词 | 搜索词与视图条件继续叠加过滤 |
| 点击「保存视图」并输入名称 | 当前条件保存到「视图」下拉框 |
| 从下拉框选择已保存视图 | 自动恢复该视图条件并刷新列表 |
| 选择下拉框默认「视图」或点击「清除条件」 | 恢复显示全部资产 |

## 与预期相比

- 达成 B3 的窄版 Dataview 目标：可以保存条件组合，覆盖「某年 + 某领域 + 某节点」的常见创作查询。
- 保守实现：没有引入完整查询语言，降低误操作和维护成本。
- 与 B2 兼容：类型条件支持「速记」，但复盘和图谱仍不被速记污染。
- 暂未实现：视图删除、视图重命名、复杂布尔语法、排序规则、跨项目查询；这些可作为 B3.1 或 B4 后续增强。
