# 自查高优先级修复完成报告

## 做了什么

### tests/analyzer.test.js
- 给 analyzer 测试文件增加块级作用域，避免和 `db.test.js` 在全局脚本里重复声明 `const test / assert / assertEqual`。
- 修复前 Chrome 只执行了 `db.test.js` 的 15 个用例；修复后 `db.test.js` 和 `analyzer.test.js` 都会执行。

### db.js
- `buildArticleMarkdown()` 增加速记分支：
  - `article.type = 'idea'` 时写出 `## 速记` 和正文。
  - 让 B2 速记可以进入 A1 本地 Markdown 文件夹。
- `parseMarkdownRecord()` 加固列表解析：
  - `文章优势`、`知识节点`、`下一步选题` 只要出现非空但格式异常的行，就不写回对应数组。
  - 避免“6 个节点只解析成功 5 个，然后静默覆盖旧 6 个”的半截覆盖问题。
- `parseMarkdownRecord()` 增加速记回读：
  - 解析 `## 速记` 段落为 `article.body`。
  - 保留标题为 `article.title`。
  - 速记正文里的 `##` Markdown 标题不会截断正文。
- `syncFromLocalFolder()` 支持速记文章补丁：
  - 对已有 `article.type = 'idea'` 的记录，允许本地 Markdown 改动回写标题和正文。
  - 非速记文章不会因为出现 `## 速记` 段落被误改类型。

### panel.js
- `saveQuickIdea()` 保存 IndexedDB 后调用 `writeArticleToLocalFolder(record)`。
- 未绑定本地文件夹时自动跳过，不影响速记主流程。

### tests/db.test.js
- 新增 3 个测试：
  - `buildArticleMarkdown: 速记写出本地 Markdown 正文`
  - `parseMarkdownRecord: 知识节点部分异常时整段不覆盖旧数据`
  - `parseMarkdownRecord: 解析速记正文为 article patch`
