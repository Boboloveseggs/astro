# 完成报告：V21 宇宙左右旋转修正

## 本次目标

修复宇宙拖拽“只能上下转，不能左右转”的问题。

## 问题原因

上一版水平拖拽主要写入了 `z` 轴 roll：

- `targetRotation.z += dx * UNIVERSE_DRAG_ROTATE`
- `targetRotation.y += dx * UNIVERSE_DRAG_ROTATE * 0.46`

这会让左右拖拽更像画面在屏幕内打滚，而不是宇宙向左/向右转身。真正的左右旋转应该主要写入 `y` 轴 yaw。

## 实际改动

### `panel.js`

- 水平拖拽改为主要控制 `y` 轴 yaw：
  - `targetRotation.y = dx * 1.08`
  - `targetRotation.z = dx * 0.18`
  - 见 `panel.js:3165-3166`。
- 保留少量 `z` 轴 roll，只作为宇宙漂移感，不再作为主旋转。
- 上下拖拽仍控制 `x` 轴 pitch。

### `tests/panel_map.test.js`

- 新增断言：水平拖拽必须主要改变 `targetRotation.y`，见 `tests/panel_map.test.js:249`。
- 新增断言：水平拖拽不能主要变成 `z` 轴 roll，见 `tests/panel_map.test.js:250`。

## 验收标准对照

| 用户反馈 | 当前结果 |
| --- | --- |
| 只能上下转 | 已修复，左右拖拽主要 yaw |
| 不能左右转 | 水平拖拽显著改变 `targetRotation.y` |
| 不要又变成屏幕内打滚 | `z` 轴 roll 只保留 18% |
