# 完成报告：项目栏抓取按钮改为面板直接注入

完成时间：2026-05-10 19:19

## 问题

项目栏小按钮仍显示「没有抓到可分析内容」。这说明按钮到 `background.js` 的消息链路没有稳定返回文章对象，继续依赖 service worker 会反复踩侧边栏 / 当前 tab / 旧 worker 缓存的问题。

## 修复内容

1. `panel.js`
   - `recaptureCurrentPage()` 改为优先由面板直接抓取：
     - 找到当前或最近的知乎标签页；
     - 先尝试 `chrome.tabs.sendMessage(tabId, { action: 'capture_request' })`；
     - 再尝试 `chrome.scripting.executeScript()` 直接注入抓取函数；
     - 抓到后由 panel 自己写入 `chrome.storage.local.lastArticle`；
     - 立刻调用 `loadArticle()` 显示新文章。
   - 只有面板直接抓取失败时，才继续走后台消息兜底。
   - 新增：
     - `captureArticleFromTabInPanel()`
     - `isCapturedArticleReady()`
     - `normalizeCapturedArticle()`
     - `grabZhihuPageContentForPanel()`

2. `tests/panel_analysis.test.js`
   - 新增测试：验证直接注入成功时不会再调用后台兜底。
   - 测试确认 panel 会自己写入 `lastArticle` 并载入新正文。
