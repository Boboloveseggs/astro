# 问题日志完成报告 · Claude 审核闸门

## 做了什么

### 新增问题日志
- 新建 `问题日志_Claude审核清单.md`，作为后续每次改动前必须查看的风险清单。
- 把已经暴露过的问题按 P0 / P1 / P2 分类记录。
- 标注每个问题的当前状态：已修、未修、可接受。
- 为每类问题写入固定审核点，避免后续重复踩坑。

### 固定后续工作流
- 每次开工前先读问题日志。
- 每次改动前确认影响面和不动清单。
- 每次完成后生成 `完成报告_*.md`。
- 完成报告必须回答：是否触碰数据、是否可能覆盖旧数据、是否维护 `articles` 与 `nodes` 一致性、是否改变 schema 或格式、如何验收。

### 纳入 Claude 三天后审核标准
- 完成报告标题必须能看出改动主题。
- 完成报告必须能让三天后的 Claude 独立理解、复现、审查。
- 数据层改动默认按 P0 风险处理。
- 没有自动化测试时，必须写清手动验收步骤。
