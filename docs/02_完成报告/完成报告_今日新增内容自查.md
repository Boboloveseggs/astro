## 发现的问题

### P0 候选：测试页实际只跑了 15 个用例

- 位置：
  - `tests/test.html:30`
  - `tests/test.html:31`
  - `tests/db.test.js:5`
  - `tests/analyzer.test.js:5`
- 现象：`db.test.js` 和 `analyzer.test.js` 都在全局脚本里声明 `const { test, assert, assertEqual } = Suite;`。
- Chrome headless 控制台报错：`Uncaught SyntaxError: Identifier 'test' has already been declared`。
- 结果：页面底部显示 `15 passed / 0 failed`，不是完成报告里写的 `24 passed / 0 failed`。
- 风险：A4 测试报告目前偏乐观，`analyzer.test.js` 的 9 个用例没有被实际执行。

### P0 候选：本地同步仍可能发生“部分解析覆盖”

- 位置：
  - `db.js:552`
  - `db.js:574`
  - `db.js:579`
  - `db.js:656`
  - `db.js:658`
- 现象：如果 `## 知识节点` 里有 6 行，其中 5 行格式正确、1 行格式异常，当前逻辑会把 5 个解析成功的节点写回 `analysis.nodes_hit`。
- 风险：用户原本有 6 个节点，只因 1 行括号或破折号格式被改坏，就可能静默丢 1 个节点。
- 建议：列表段落只要存在非空但解析失败的行，就不要写回该数组；或者把异常显式返回给同步结果。

### P1：灵感快捕没有进入本地 Markdown 真相源

- 位置：
  - `panel.js:261`
  - `panel.js:287`
  - `panel.js:251`
  - `panel.js:253`
  - `db.js:501`
  - `db.js:518`
- 现象：正式文章保存后会调用 `writeArticleToLocalFolder(record)`，速记只调用 `dbSaveArticle(record)`。
- 风险：B2 速记只在 IndexedDB 里，本地文件夹双向同步看不到它；这和“本地优先 / Markdown 是真相源”的方向不完全一致。
- 补充：即使后续给速记调用 `writeArticleToLocalFolder()`，`buildArticleMarkdown()` 目前也没有速记正文分支，需要一起补。

### P1/P2：Dataview 保存视图不进入 JSON 备份

- 位置：
  - `panel.js:466`
  - `panel.js:592`
  - `db.js:192`
  - `db.js:194`
- 现象：B3 保存视图写入 `config.asset_views`，但 `dbExportAll()` 只导出 `articles` 和 `nodes`。
- 风险：用户做了很多保存视图后，如果用 JSON 备份/恢复，视图会丢失。
- 建议：后续备份格式加入可迁移的 `config` 白名单，例如 `projects`、`asset_views`，不要包含 `local_folder` 这种不可序列化句柄。

### P2：速记入口在检测到文章时不可见

- 位置：
  - `panel.html:565`
  - `panel.html:570`
  - `panel.js:140`
  - `panel.js:158`
- 现象：速记入口放在 `waitingSection` 里；检测到文章后 `waitingSection` 被隐藏。
- 风险：用户打开知乎文章时想快速记一个想法，要先切手动或重置，和“30 秒速记”有一点冲突。

### P2：地图 simulation 离开地图 Tab 后没有停止

- 位置：
  - `panel.js:440`
  - `panel.js:450`
  - `panel.js:786`
  - `panel.js:787`
  - `panel.js:967`
  - `panel.js:974`
- 现象：`renderMapView()` 会停止旧 simulation，但 `switchTab()` 离开地图时不会停止；地图节点不足直接返回时也不会停止旧 simulation。
- 风险：隐藏图谱可能继续 tick，属于性能和体验边角问题。

### P2：知识球层级是“角色优先级”，不是“最高频角色”

- 位置：
  - `panel.js:744`
  - `panel.js:745`
  - `panel.js:746`
- 现象：只要某节点出现过一次 `primary`，就会被放入主节点圈，即使它更多时候是 `secondary` 或 `background`。
- 风险：如果用户理解为“根据出现频次决定层级”，当前实现会有偏差。
