# B4 完成报告 · 跨项目节点库入口

## 做了什么

### panel.js
- 在资产页增加 `assetListMode`，区分：
  - `assets`：文章 / 速记资产列表。
  - `nodes`：全局知识节点库。
- 资产工具栏新增「节点」按钮。
- 新增 `setAssetListMode(mode)`：
  - 切换「节点」按钮高亮。
  - 根据模式切换搜索框 placeholder。
- 新增 `buildNodeIndex(articles, projectNameById)`：
  - 从全部文章的 `analysis.nodes_hit` 汇总全局节点。
  - 统计每个节点出现过的作品数、项目数、角色分布。
  - 按跨项目数量、作品数量、出现次数排序。
- 新增 `loadNodeIndex()`：
  - 在资产 Tab 中显示节点库。
  - 支持用搜索框搜索节点名、节点类型、项目名。
  - 点击节点进入该节点的跨项目反链详情。
- `showNodeDetail(nodeName, backTarget)` 增加来源参数：
  - 从节点库进入时，返回按钮显示「← 节点库」。
  - 从文章卡片或地图进入时，仍返回「全部资产」。
- 本地文件夹重新扫描后，如果当前停留在节点库，会刷新节点库而不是强制切回资产列表。

### panel.html
- 新增节点库列表样式：
  - `.node-index-meta`
  - `.node-project-row`
  - `.node-project-pill`
- 给 `.assets-toolbar-btn.active` 增加高亮样式。

### 问题日志_Claude审核清单.md

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 打开「资产」Tab | 顶部工具栏出现「节点」按钮 |
| 点击「节点」 | 资产列表切换为全局节点库 |
| 查看节点卡片 | 显示节点名、类型、角色分布、作品数、项目数 |
| 一个节点出现在多个项目 | 节点卡片显示多个项目名，项目数大于 1 |
| 在搜索框输入节点名 | 节点库只显示匹配节点 |
| 在搜索框输入项目名 | 节点库只显示该项目相关节点 |
| 点击节点卡片 | 进入节点详情，显示该节点跨项目引用的全部作品 |
| 在节点详情点击返回 | 从节点库进入时返回节点库 |
| 点击反链作品卡片 | 进入对应资产详情 |

## 与预期相比

- 达成 B4 的可发现入口：节点反链不再只能从文章详情或地图里“碰到”，资产页现在有固定节点库入口。
- 没有新增 DB schema，跨项目关系直接从已有 `analysis.nodes_hit` 和 `project_id` 汇总。
- 保留原节点详情能力：仍显示出现作品数、项目数、角色统计和反链作品。
- 暂未实现节点详情的独立 URL、节点重命名、节点合并；这些更适合放到 C 阶段编辑能力。
