# A3 完成报告 · 标签系统

## 做了什么

### analyzer.js
- prompt 新增 `"tags"` 字段，要求 AI 返回 3-5 个不超过 6 字的简短标签，用于快速归类过滤

### db.js · `buildArticleMarkdown()`
- 导出 `.md` 文件时，在领域/视角字段下方追加 `**标签**：xxx · xxx · xxx` 一行

### panel.html
- 新增 `.asset-tag-row`、`.asset-tag`、`.asset-tag.active` 样式（深色圆角小标签，悬停变色，激活状态有蓝色边框）

### panel.js
- **搜索过滤**：`loadAssets()` 过滤字段加入 `a.tags`，搜索词可以直接命中标签
- **资产卡片**：在核心判断下方渲染标签行，当前激活的搜索词与标签文字完全匹配时高亮显示
- **标签点击**：点击标签 → 把标签文字填入搜索框并重新过滤；再次点击同一标签 → 清空搜索框（取消过滤）
- **分析结果视图**：新增 `cardTags(r)` 函数，在定位卡片之后、优势之前显示标签卡片
- `renderResult()` 调用顺序更新：`cardPosition → cardTags → cardStrengths → …`

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 分析一篇新文章 | 分析结果里出现「标签」卡片，显示 AI 返回的 3-5 个标签 |
| 打开「资产」Tab，查看已有记录（新分析的）| 卡片底部显示标签行 |
| 点击某个标签 | 搜索框自动填入该标签，列表过滤为只含该标签的文章，激活标签高亮 |
| 再次点击同一标签 | 搜索框清空，恢复显示全部记录 |
| 手动在搜索框输入标签关键词 | 也能命中含该标签的文章 |
| 旧文章（无 tags 字段）| 卡片底部没有标签行，不报错 |
