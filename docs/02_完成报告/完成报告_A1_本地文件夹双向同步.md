# A1 完成报告 · 本地文件夹双向同步

## 做了什么

### db.js
- 新增 `parseMarkdownRecord(text, filename)` — 把扩展写出去的 `.md` 文件反向解析成 analysis 对象，覆盖领域、视角、核心判断、知识节点、选题建议、编辑评语等所有字段
- 新增 `syncFromLocalFolder()` — 遍历已绑定文件夹里所有 `.md` 文件，按 URL 或文件名末尾 shortId 匹配现有记录，把解析后的 analysis 合并覆盖写回 IndexedDB，返回同步数量

### panel.html
- 把 `<div id="assetsTab">` 拆成两层：`assetsToolbar`（持久工具栏）+ `assetsList`（动态列表）
- 新增工具栏样式 `.assets-toolbar`、`.assets-toolbar-btn`、`.sync-status`

### panel.js
- 新增 `initAssetsToolbar()` — 启动时检查是否绑定了文件夹，有则在资产栏顶部渲染「重新扫描本地文件夹」按钮
- 新增 `syncLocalFolder()` — 按钮点击后显示「扫描中…」，完成后显示「已同步 N 条」或错误信息，同步有变化则自动刷新资产列表
- `loadAssets()` 和 `showAssetDetail()` 的写入目标改为 `assetsList`，与工具栏分离，互不干扰

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 在设置页绑定本地文件夹，分析一篇文章 | 文件夹里生成一个 `.md` 文件 |
| 用系统文件管理器打开该文件，手动修改「编辑评语」那段文字，保存 | 文件内容已变更 |
| 回到扩展面板 → 资产 Tab → 点击「重新扫描本地文件夹」 | 显示「已同步 1 条」 |
| 打开该条资产详情 | 编辑评语字段显示手动改过的内容 |
| 未绑定文件夹时 | 工具栏区域为空，不出现扫描按钮 |
