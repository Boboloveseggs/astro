# 完成报告：V21 锁定星球点阵预览

## 本次目标

修正锁定知识星球后的视觉：不要再显示突兀的紫色实心大球，而是在宇宙中先展示一个由该知识球节点组成的点阵球预览；再次点击后才进入可操作的金色知识球。

## 实际改动

### `panel.js`

- 新增 `collectUniversePreviewNodes(ball)`，从当前知识球的文章里统计真实节点、命中次数和 primary 角色，见 `panel.js:2357`。
- 新增 `universeKnowledgePreviewTexture(ball)`，生成透明 Canvas 贴图：
  - 使用 768px 高分辨率贴图，避免拉近后糊。
  - 用真实节点作为亮点，节点命中越多越亮。
  - 数据少时补充低亮度结构点，让它仍读得出球体形态。
  - 去掉实体球面和硬边框，只保留淡雾、边缘微光点和节点光点。
  - 标记 `userData.kind = 'knowledge-preview'`，见 `panel.js:2396`、`panel.js:2495`。
- 新增 `ensureUniversePreviewTexture(sprite)`，按需生成并缓存该知识星球的预览贴图，见 `panel.js:2503`。
- 锁定态渲染改为使用点阵预览贴图：
  - 动画循环中选中星球使用 `ensureUniversePreviewTexture(sprite)`，见 `panel.js:3082`。
  - 初始自动锁定时也使用同一预览贴图，见 `panel.js:3283`。
  - 锁定预览尺寸从 10:13 的实心行星大尺寸收回到 `Math.max(88, baseScale * 2.20)`，见 `panel.js:3095`、`panel.js:3288`。

### `tests/panel_map.test.js`

- 将锁定星球测试从“使用 planetTexture”改为“使用 knowledge-preview 预览贴图”，并断言预览里有节点数量，见 `tests/panel_map.test.js:209-211`。

## 验收标准对照

| 用户要求 | 当前结果 |
| --- | --- |
| 紫色大球太突兀 | 已移除实体紫色球面 |
