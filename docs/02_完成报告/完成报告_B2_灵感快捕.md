# B2 完成报告 · 灵感快捕

## 做了什么

### panel.html
- 在「分析」Tab 的等待区新增速记入口：
  - `quickTitleInput`：灵感标题
  - `quickBodyInput`：灵感正文
  - `quickSaveBtn`：保存速记
  - `quickStatus`：保存状态提示
- 新增速记相关样式：
  - `.quick-capture`
  - `.quick-input`
  - `.quick-row`
  - `.quick-status`
  - `.btn-secondary`
- 新增速记详情样式：
  - `.idea-detail-title`
  - `.idea-detail-meta`
  - `.idea-detail-body`

### panel.js
- 新增 `saveQuickIdea()`：
  - 不调用 AI。
  - 不写 `nodes` 表。
  - 不改变 IndexedDB schema。
  - 复用现有 `articles` store，保存为 `article.type = 'idea'` 的轻量资产。
- 资产搜索范围加入 `rec.article.body`，速记正文可被搜索。
- 资产列表支持 `idea` 类型：
  - 类型显示为「速记」。
  - 卡片摘要显示速记正文片段。
  - 空状态文案从「还没有分析记录」调整为「还没有资产」。
- 新增 `showIdeaDetail(rec)`：
  - 速记详情显示标题、日期、正文。
  - 支持删除单条速记。
- `loadReview()` 排除速记，只统计已分析文章，避免速记污染复盘数据。

### db.js
- `buildObsidianMarkdown()` 支持速记导出：
  - front-matter 保留 `type: "idea"`。
  - 正文输出 `## 速记` 段落。

### tests/db.test.js
- 新增测试：`buildObsidianMarkdown` 能把速记正文写入 Markdown。

### 完成报告_A4_最小单元测试.md
- 更新测试数量：`db.test.js` 从 14 个用例变为 15 个，总测试数从 23 个变为 24 个。

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 打开「分析」Tab | 手动粘贴区下方出现速记输入区 |
| 只填写速记正文并点击「保存速记」 | 显示「已保存到资产库」，资产数量增加 |
| 不填写正文直接保存 | 显示「先写一点内容」，不新增资产 |
| 打开「资产」Tab | 速记以「速记」类型出现在列表里 |
| 在资产搜索框输入速记正文关键词 | 能搜到对应速记 |
| 点击速记卡片 | 进入速记详情页，显示标题、日期、正文 |
| 删除速记 | 资产数量减少，列表刷新 |
| 打开「复盘」Tab | 速记不计入已分析文章统计 |
| 导出 Obsidian Vault | 速记 Markdown 中包含 `## 速记` 和正文 |
| 打开 `chrome-extension://[ID]/tests/test.html` | 底部显示最新测试总数通过；当前为 `37 passed / 0 failed` |

## 与预期相比

- 达成 B2 目标：30 秒内可以记录一个想法，不必走 AI 分析。
- 保守实现：速记复用现有 `articles` store，没有新增 DB schema，也没有新增迁移风险。
- 与产品方向一致：速记是创作者工作台里的轻量资产，不是普通通用笔记。
- 暂未实现：速记标签、速记转正式分析、速记与节点的 Wikilink 自动关联；这些更适合放到 B3 Dataview 或 C2 Wikilink 阶段。
