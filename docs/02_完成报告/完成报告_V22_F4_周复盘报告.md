## 风险与回滚

- 风险：后台周日晚自动生成依赖浏览器扩展运行、API Key 可用、网络可用；失败时会等用户进入复盘页再前台兜底。
- 风险：当前周报按当前项目数据生成，但缓存 key 按自然周保存；多项目深度使用时后续可升级成项目维度缓存。
- 回滚路径：移除 `alarms` 权限、删除 background 周报逻辑、删除复盘页周报卡和 `weeklyReports` 读写函数。
