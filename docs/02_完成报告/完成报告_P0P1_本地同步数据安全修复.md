# P0/P1 完成报告 · 本地同步数据安全修复

## 做了什么

### db.js
- 新增 `dbReplaceArticleNodes(nodesHit, articleId)`：本地 Markdown 反向同步解析到 `nodes_hit` 时，先移除该文章在旧节点里的引用，再写入新节点引用。
- 调整 `parseMarkdownRecord(text, filename)`：只返回成功解析到的字段，避免解析失败的字段用空字符串或空数组覆盖 IndexedDB 里的旧数据。
- 调整 `syncFromLocalFolder()`：只有解析出可信 `analysis` 字段时才写回；如果本次同步包含可信 `nodes_hit`，同步更新 `nodes` 表，保证地图和复盘数据一致。

### panel.js
- 统一本地扫描按钮文案：初始状态和同步结束后都显示「重新扫描」，避免按钮文字变长挤压状态文字。

### tests/db.test.js
- 新增回归测试：当「知识节点」段落格式异常时，`parseMarkdownRecord()` 不返回 `nodes_hit: []`，从而保留旧节点数据。

### 完成报告_A4_最小单元测试.md
- 更新测试数量：`db.test.js` 从 11 个用例变为 12 个，总测试数从 20 个变为 21 个。

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 在本地 `.md` 里只修改「编辑评语」并保持其他段落不变 | 点击「重新扫描」后，资产详情里的编辑评语更新 |
| 把 `.md` 里的知识节点格式误改坏，例如把全角括号改成半角括号 | 点击「重新扫描」后，旧的 `nodes_hit` 不会被空数组覆盖 |
| 在 `.md` 里按正确格式新增或删除知识节点 | 点击「重新扫描」后，地图和复盘里的节点频次随之变化 |
| 点击资产页「重新扫描」按钮 | 扫描前后按钮宽度保持稳定，不挤压状态文字 |
| 打开 `chrome-extension://[ID]/tests/test.html` | 底部显示 `21 passed / 0 failed` |

## 与预期相比

- 达成 P1：扫描按钮文案不再闪变。
