# 完成报告：V22 F3 周目标可设

时间：2026-05-09

## 本次改动

- `options.html`
  - 新增“创作目标”设置区。
  - 新增 `周创作目标` 数字输入框，默认值 3，范围 1-99。
  - 增加说明文案：`每周想写几篇？球球会按这个数字给你看进度。`

- `options.js`
  - 新增 `normalizeWeekGoal()`，统一限制目标范围。
  - `loadSettings()` 读取 `chrome.storage.local.weekGoal`，无值时默认 3。
  - `saveSettings()` 保存 `weekGoal` 到 `chrome.storage.local`。

- `panel.js`
  - 顶部状态条读取 `chrome.storage.local.weekGoal`。
  - 本周进度文案改为 `本周 N/M 篇`。
  - 达成目标时文案改为 `本周目标达成 ✓`，进度条进入完成态。

- `panel.html`
  - 本周进度默认文案从 `0/5` 改为 `0/3`。
  - 新增 `.prog-wrap.achieved` 金色完成态样式。

- `tests/panel_assets.test.js`
  - 新增周目标归一化测试。
  - 更新状态条测试，验证读取 `weekGoal`。
  - 新增目标达成测试，验证完成文案与完成态 class。

## 验收标准

- 设置页能设置每周想写几篇，默认 3 篇。
- 设置持久化到 `chrome.storage.local.weekGoal`。
- 顶部进度显示 `本周 N/M 篇`，M 使用用户设置值。
- 达成目标时显示 `本周目标达成 ✓`，进度条为金色完成态。
- 不改 schema，不改 Markdown/JSON 格式，不碰宇宙视觉。

## 实际结果

- 已符合：设置页新增周创作目标。
- 已符合：状态条读取本机 `weekGoal`。
- 已符合：目标达成后显示完成文案与金色完成态。

## 验证记录

- 前端测试：`102 passed / 0 failed`
- 新增/更新测试：
  - `normalizeWeekGoal: 周目标默认 3 篇并限制在合理范围`
  - `updateTopMetrics: 渲染五列状态条和本周进度`
  - `updateTopMetrics: 本周目标达成时显示达成文案`

## 风险与回滚

- 风险：设置页的“保存设置”现在同时保存 API 设置和周目标；如果用户只改目标也要点保存。
- 回滚路径：移除 `weekGoal` 输入框、恢复 `DEFAULT_WEEK_METRIC_GOAL` 的固定值逻辑，并删掉新增测试。
