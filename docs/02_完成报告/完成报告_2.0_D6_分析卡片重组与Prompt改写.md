# 完成报告 2.0 D6：分析卡片重组与 Prompt 改写

完成时间：2026-05-08  
执行依据：用户 D6 指令 + `任务下发_2.0冲刺_九日清单.md` T-5.2 / T-5.3 / T-5.4  
性质：AI prompt 层 + 分析结果 UI 层；不涉及 IndexedDB schema、迁移或数据写入逻辑

## 做了什么

1. `analyzer.js` 的 prompt JSON schema 增加 4 个新字段：
   - `creator_strength`
   - `craft_review`
   - `reusable_clips`
   - `essence_insights`
2. `parseAndValidate()` 保持向后兼容：新字段不是必填，旧分析记录缺少这 4 个字段不会报错。
3. 分析结果页从散卡重组为 5 个创作流水线分组：
   - 定位
   - 创作者反馈
   - 素材资产
   - 延伸方向
   - 追问 AI
4. 新增卡片：
   - `cardCreatorStrength()`
   - `cardCraftReview()`
   - `cardClips()`
   - `cardEssenceInsights()`
5. 保留旧卡片并重新归位：
   - `cardPosition()`
   - `cardStrengths()`
   - `cardConcepts()`
   - `cardConnections()`
   - `cardSuggestions()`
   - `cardMapPosition()`
   - `cardNodes()`
   - `cardInsight()`
6. `groupQA()` 先保留空的 `analysis-chat-slot`，给后续 D8 追问对话接入，不做 LLM 多轮调用。
7. 分析调用的 `maxTokens` 从默认 2500 提升到 3500，给新增字段留输出空间，降低 JSON 被截断风险。
8. 新增 D6 自动化测试，覆盖 prompt、新字段解析、旧记录兼容和 5 组 UI 渲染。

## 风险预警

1. 本次只做 prompt 与渲染，真实 LLM 是否稳定返回 4 个新字段，需要后续用实际 API Key 试跑。
2. 新字段不是必填，这是为了兼容旧记录；如果模型漏返回，新卡片会自动跳过，不会阻断入库。
3. `reusable_clips.content` 要求 30 字内引用原文，但这是 prompt 约束，前端没有强截断，避免误伤中文内容。
4. D7 如果把 `reusable_clips` / `essence_insights` 入 `clips` store，要重新按 P0 级别审查 schema migration。
