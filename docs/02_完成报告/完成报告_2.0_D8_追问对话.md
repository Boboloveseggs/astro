# 2.0 D8 完成报告：追问对话

执行依据：用户 D8 指令 + `任务下发_2.0冲刺_九日清单.md` Day 8 追问对话区。

## 做了什么

1. 在分析结果和资产详情页接入「追问 AI」对话卡，提供 6 个模板按钮和自定义输入。
2. 新增独立 `isChatting` 状态，不与 `isAnalyzing` 互锁；追问请求进行中会禁用当前对话区按钮，`finally` 恢复状态。
3. 每轮 LLM 请求带入：文章正文、分析 JSON 摘要、最近 6 条对话历史、当前问题。
4. 加了上下文长度控制：正文最多 7000 字、分析摘要最多 3200 字、历史单条截断，避免多轮对话无限膨胀。
5. 模板入库规则已落地：
   - 「我这篇最好的地方是什么？」自动保存为 `立意`。
   - 「这篇里哪些素材未来可以怎么复用？」拆分后自动保存为多条 `素材`。
   - 其他模板显示「保存到素材库 / 保存到立意库」按钮。
   - 自定义追问显示「存为素材 / 存为立意 / 不存」。
6. 模板自动入库使用确定性 clip id；重复点击同一模板会覆盖同一组记录，不会新增重复素材。素材复用模板还会先清理同模板旧条目，避免上一轮多出来的旧素材残留。
7. 新分析入库时，`article.body` 从原来的前 2000 字改为保存完整正文；发送给 LLM 时再截断，兼顾追问上下文和 token 控制。

## 风险预警

1. 新记录开始保存完整正文，JSON 备份里的 `article.body` 内容会比以前更长；字段结构不变，但备份体积会增加。
2. 旧记录如果历史上只保存了前 2000 字，资产详情页追问只能用旧记录已有正文，无法凭空恢复完整原文。
3. 本轮自动化测试 mock 了 LLM 调用，没有真实消耗 API；真实模型输出的素材拆分质量仍需人工试用校准。
4. 追问 session 保存在 panel 当前运行内存中，关闭 panel 后对话历史不会持久化；本轮只沉淀用户选择保存或模板自动保存的 clip。

## Claude 8 问

1. 是否触碰数据写入、删除、导入、同步或迁移？  
   触碰写入和删除：追问回答可写入 `clips`；reuse 模板重复入库前会删除同模板旧自动素材。未触碰导入、同步、迁移。

2. 是否可能造成旧数据被覆盖、丢失、重复、孤立引用？  
   不会覆盖旧文章。模板自动 clip 使用 `chat_{articleId}_{template}` 稳定 id，重复模板会更新同一条或同一组；reuse 只删除同一项目中同一文章同一模板前缀的旧自动素材。

3. 是否维护 `articles` 与 `nodes` 一致性？  
   本轮不改 `nodes_hit`，不写 nodes 表；文章入库仍走原有 `dbReplaceArticleNodes()`。

4. 是否改变旧 Markdown 格式或 JSON 备份格式？  
   Markdown 格式不变。JSON 备份字段不变，但新文章的 `article.body` 会保存完整正文，内容长度变化已在风险预警说明。

5. 是否改变 IndexedDB schema？  
   没有。`DB_VERSION` 仍为 4，未新增 store/index。

6. 是否影响 A1-A5 已有能力？  
   A1 本地同步、A2 搜索、A3 标签、A4 测试、A5 Vault 导出均未改主流程。完整正文会让未来搜索/追问上下文更充分。

7. 自动化测试覆盖什么？不能覆盖的手动验收是什么？  
   自动化覆盖 UI 模板数、无固定 id、上下文截断、历史长度、自动入库、去重清理、防并发。手动验收建议：真实 API Key 下连续点 3 次模板，确认只发 1 个请求；模板回答出现后切资产 Tab 看素材/立意是否出现。

8. 三天后 Claude 只看完成报告，是否能复现验收？  
   可以。报告列出了文件、行号、测试文件、测试结果和手动验收路径。
