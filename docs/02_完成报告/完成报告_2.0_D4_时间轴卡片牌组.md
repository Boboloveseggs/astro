## 做了什么

1. 在复盘 Tab 增加 1.0 风格的时间轴 deck 视图，默认替代旧简陋时间线展示。
2. 增加上轴面包屑：年 / 月 / 周 / 日 / 当前作品。
3. 增加下轴圆点时间线：日期圆点、金色横线、active 放大和金色光晕。
4. 增加横向卡片牌组：每篇作品一张卡，含标题、标签、摘要、查看全文入口。
5. 增加发牌动画 `tlDealIn`：卡片从右侧 36px、旋转 3deg 飞入，按 `--deal-index` 错峰进入。
6. 增加底部 slider 与左右切换按钮：拖动 slider 或点左右按钮会切换 active 卡片。
7. 增加 430ms 平滑居中滚动：切换卡片时，当前卡自动滚到牌组中间。
8. 增加右侧日期 gear 列表：可按日期切换牌组。
9. 增加「卡片视图 / 列表视图」切换：旧 `reviewTimeline()` 保留为列表兜底，默认使用新 deck。
10. 增加 D4 专项测试和 fixture 截图页。

## 适配点

1. 不是一字不差照搬 1.0 CSS。v4 扩展面板宽度更窄，所以右侧 gear 从固定 260px 改成 `clamp(118px,28vw,260px)`，保留 260px 上限，并增加 520px 以下堆叠布局。
2. 1.0 的 inline 行为在 v4 中改为 `addEventListener` 绑定，避免 Chrome MV3 CSP 风险。
3. 1.0 的专用字段替换为 v4 数据结构：`rec.article.title`、`rec.article.body`、`rec.savedAt`、`rec.analysis.domain`、`rec.analysis.sub_domain`、`rec.analysis.tags`、`rec.analysis.nodes_hit`。
4. 旧 `reviewTimeline()` 没有删除，而是保留为「列表视图」兜底。Day 5 功能清理日再决定是否彻底删除或合并。
