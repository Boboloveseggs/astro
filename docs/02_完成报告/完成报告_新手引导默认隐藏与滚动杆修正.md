# 新手引导默认隐藏与滚动杆修正完成报告

完成时间：2026-05-11

## 本次目标

修正两个新手引导体验问题：

- 第 13 步出现难看的白色拖动杆。
- 第一次打开面板时不要自动弹出新手引导；引导应一直藏在右上角问号里，由创作者自己选择是否进入教程。

## 实际改动

### 1. 取消首次自动弹出新手引导

文件：`panel.js`

- `maybeStartPanelOnboarding()` 现在只负责绑定引导按钮事件。
- 不再读取 `onboardingSeen` 来判断首次打开。
- 不再在首次打开时调用 `startPanelOnboarding(0)`。
- 创作者只有点击右上角 `?` 才会进入新手引导。

这样默认体验变成：

- 想自由探索的创作者：打开后直接使用产品。
- 想看教程的创作者：点击问号进入引导。

### 2. 问号入口仍然可继续引导

文件：`panel.js`

- `replayPanelOnboarding()` 改为异步函数。
- 如果之前从 API 设置页留下了续接位置，点击问号时会从续接位置继续。
- 没有续接位置时，从第 1 步开始。

### 3. 修正第 13 步白色拖动杆

文件：`panel.html`

- `.onboarding-card` 隐藏滚动条：
  - `scrollbar-width: none`
  - `-ms-overflow-style: none`
  - `::-webkit-scrollbar { width: 0; height: 0; }`
- `.onboarding-layer.compact .onboarding-card` 不再使用内部滚动：
  - `max-height: none`
  - `overflow: visible`

这样第 13 步这种 compact 全貌卡不会再露出系统白色滚动块。

### 4. 增加防回归测试

文件：`tests/panel_cleanup.test.js`

- 新增测试：
  - `maybeStartPanelOnboarding: 首次打开不自动弹出，引导只藏在问号入口`
- 更新手动重看测试为 `await replayPanelOnboarding()`。
