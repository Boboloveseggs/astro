# 完成报告：项目栏重新抓取当前知乎页按钮

完成时间：2026-05-10 18:05

## 本次目标

在「默认项目」项目选择旁边增加一个小的知识图线按钮。用户不固定浏览器扩展图标时，也能在侧边栏里点击按钮，重新抓取当前知乎页面，并回到分析页准备分析新文章。

## 改动内容

1. `panel.html`
   - 在 `projectSelect` 旁新增 `recaptureBtn` 小按钮。
   - 按钮使用三点连线的图线图标，无文字，不挤占主界面。
   - 增加加载态 `is-loading`，点击抓取时有轻微呼吸反馈。

2. `panel.js`
   - 初始化时绑定 `recaptureBtn` 点击事件。
   - 新增 `recaptureCurrentPage()`：
     - 点击后切回分析页；
     - 向后台发送 `capture_current_tab`；
     - 后台抓到文章后调用 `loadArticle()`，显示检测到的新文章；
     - 抓取失败时在分析页显示错误提示；
     - 请求结束后恢复按钮状态。

3. `background.js`
   - 保留原有「点击扩展图标 -> 打开 panel -> 抓知乎文章」能力。
   - 新增 `capture_current_tab` 消息入口，复用同一个 `captureFromTab()` 抓取函数。
   - `captureFromTab()` 现在会返回 `{ ok, article/error }`，方便 panel 得到明确结果。

4. `tests/panel_analysis.test.js`
   - 新增按钮抓取测试：确认点击逻辑会请求后台、切回分析页、载入新文章正文和标题。
   - 顺手补齐该测试文件已使用的 `assertEqual` 解构，避免测试文件自身存在隐性依赖。
