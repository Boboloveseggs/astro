# 节点反链详情完成报告 · 内嵌 Obsidian 核心能力

## 做了什么

### panel.js
- 将知识节点从静态展示升级为可点击对象：
  - 分析结果页里的「知识节点」可点击。
  - 资产详情页里的「知识节点」可点击。
  - 地图 Tab 里的 D3 节点可点击。
- 新增 `showNodeDetail(nodeName)`：
  - 读取全部文章和项目。
  - 按节点名匹配所有引用该节点的作品。
  - 显示该节点出现于多少篇作品、多少个项目。
  - 汇总该节点在作品中的角色分布：主节点、次节点、背景节点。
  - 列出所有反向引用作品，点击作品卡片可进入资产详情。
- 调整 `switchTab(name, opts)`：支持 `skipLoad`，避免打开节点详情时被资产列表异步刷新覆盖。
- 地图节点增加点击事件：点击图谱中的任意节点，直接进入该节点的反链详情。
- 补强 `esc()`：增加 `"` 和 `'` 的转义，保证节点名、标签等内容进入 HTML 属性时更安全。

### panel.html
- 新增节点 hover 状态，让可点击节点有明确反馈。
- 新增节点详情头部样式：
  - 节点标题
  - 类型 badge
  - 作品数量
  - 项目数量
  - 角色分布
- 新增反链作品卡片的辅助间距样式。

## 验收标准

| 步骤 | 预期结果 |
|------|----------|
| 打开某篇资产详情 | 「知识节点」卡片里的每个节点有 hover 反馈 |
| 点击某个知识节点 | 自动切到「资产」Tab，并显示该节点详情 |
| 查看节点详情顶部 | 能看到节点名、节点类型、关联作品数量、关联项目数量、角色分布 |
| 点击节点详情里的作品卡片 | 进入对应资产详情页 |
| 进入「地图」Tab 并点击图谱节点 | 进入同一个节点反链详情页 |
| 点击「← 全部资产」 | 返回资产列表 |
| 节点名中包含引号等特殊字符 | 不破坏页面结构 |

## 与预期相比

- 符合你澄清后的方向：不是内嵌 Obsidian，而是把 Obsidian 的核心优势内化为产品能力。
- 本次实现的是最窄但最关键的一块：反链。节点现在不只是图谱里的点，而是作品之间的连接入口。
- 比单纯 Obsidian 导出更进一步：这个能力在知境图鉴内部直接可用，不依赖外部 Obsidian。
- 暂未实现 Dataview 查询语法、节点别名、Wikilink 自动补全；这些可以建立在本次节点详情之上继续做。
