# 完成报告：V21 宇宙体积感与自由旋转修正

## 本次目标

解决归档宇宙“像平面沙盘、不够立体、翻转到一定角度会被阻碍”的问题。目标是让宇宙本体具有真实的 3D 厚度，拖拽时能自由俯仰旋转，而不是一张平面上摆着星球。

## 实际改动

### `panel.js`

- 放开宇宙拖拽俯仰限制：
  - 新增 `normalizeUniverseAngle()`，让旋转值可持续绕圈但不会无限累积，见 `panel.js:1707`。
  - 移除原来的 `x` 轴夹角限制，拖拽不再被 `-0.92 ~ 0.62` 卡住，见 `panel.js:3103-3105`。
- 增强知识星球的真实 3D 分布：
  - `universe3DPoint()` 的 `z` 深度从薄层扩大到约 `±160`，并增加纵向分布，见 `panel.js:2734`。
- 增加宇宙体积粒子层：
  - `createUniversePointCloud()` 新增 `halo` 模式，生成包围星云的三维粒子云，见 `panel.js:2771`。
  - 渲染时加入 `haloCloud = createUniversePointCloud(3800, 'halo', pointTexture)`，见 `panel.js:3296`、`panel.js:3305`。
- 加厚原有星云粒子：
  - `haze / dust / bright` 粒子的 y 厚度、z 厚度和波动纵深都增加，避免侧翻时成为薄片。
  - `createUniverseSpiralArmCloud()` 的螺旋臂 z 纵深增加，见 `panel.js:2838`。
- 降低平面底图的主导性：
  - `createUniverseGalaxyPlane()` 透明度从 `0.44` 降为 `0.24`，见 `panel.js:2899`。
  - 平面底图侧翻时根据朝向自动淡出，避免暴露“平板感”，见 `panel.js:3171-3172`。

### `tests/panel_map.test.js`

- 新增 3D halo 粒子层断言，见 `tests/panel_map.test.js:217`。
- 新增螺旋星云 z 纵深断言，见 `tests/panel_map.test.js:222`。
- 新增竖向拖拽不被夹角限制的断言，见 `tests/panel_map.test.js:248`。

## 验收标准对照

| 用户要求 | 当前结果 |
| --- | --- |
| 不要像一个沙盘 | 增加 3D halo 粒子层，星云不再只靠平面底图成立 |
| 翻转后不要露出平面 | 平面底图降低透明度，并在侧翻时自动淡出 |
| 宇宙要有立体感 | 星球、螺旋臂、dust/haze/bright 粒子全部增加 z 纵深 |
| 拖拽不要受阻 | 移除 x 轴夹角钳制，支持更深的俯仰旋转 |
