# V21 T8 粒子星云真实感增强完成报告

生成时间：2026-05-09

## 本次目标

进一步修改地图页的宇宙 / 星云视觉。去掉“几条发光面条拼在一起”的感觉，让星云更像由大量不同亮度、不同大小、不同冷暖色的星点构成。整体方向是更华丽、更真实、更有密度。

## 改动内容

1. `panel.js`
   - 新增 `hashSigned()` 和 `hashNormal()`，用于生成稳定的随机散布和星云厚度。
   - 新增 `rgbaFromRgb()` 和 `paintGalaxyParticle()`，用径向渐变画单个星云粒子。
   - 重写 `universeGalaxyTexture()`：
     - 不再用连续 stroke 画旋臂。
     - 改为约 9000 个纹理粒子组成星云主体。
     - 粒子包含暖金、冷蓝、亮核、暗尘带等层次。
     - 给 texture 写入 `userData.particleCount`，便于测试确认不是线条贴图。
   - 强化 3D 点云：
     - `farStars`: 1200 个远景星点。
     - `hazeCloud`: 3600 个低透明度星云雾点。
     - `dustCloud`: 6200 个主体尘埃点。
     - `brightCloud`: 2200 个高亮星点。
   - 调整 `createUniversePointCloud()`：
     - 按 `far / haze / dust / bright` 四种模式生成不同分布、颜色、大小和透明度。
     - 高亮星点使用固定屏幕尺寸，保证画面里有更清晰的亮点层。
   - 放大星云平面尺寸，从 `610 x 382` 提升到 `760 x 470`。
   - 提高核心光、内光晕、外光晕亮度与范围。
   - 渲染循环中加入 haze / bright 的轻微呼吸变化。

2. `panel.html`
   - 减轻 `.universe-canvas-wrap::before` 的黑色遮罩。
   - 把遮罩混合模式从 `multiply` 改为 `normal`，避免把粒子星云压得太暗。
   - 降低扫光层 opacity，减少“条状光带”干扰。

3. `tests/universe_fixture.html`
   - 同步宇宙遮罩 CSS，保证截图验收和真实面板一致。

4. `tests/panel_map.test.js`
   - 增加粒子星云结构验收：
     - 星云纹理粒子数必须大于等于 9000。
     - haze 粒子层必须存在且大于等于 3000。
     - dust 粒子层必须存在且大于等于 6000。
     - bright 粒子层必须存在且大于等于 1200。

## 验收标准

1. 星云主体不能再依赖连续线条形成旋臂。
2. 星云应由大量点状粒子构成。
3. 星点应有亮暗、大小、冷暖色差异。
4. 宇宙背景密度要比之前更高。
5. 不改数据库 schema。
6. 不改 Markdown 格式。
7. 自动化测试全通过。
