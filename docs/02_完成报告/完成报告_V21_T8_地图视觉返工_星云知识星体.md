# 完成报告 · V2.1 T8 地图视觉返工：星云知识星体

完成时间：2026-05-08

## 一、这次要解决什么

用户反馈：地图仍然不像“宇宙”，上一版只是把金色球体贴在星云背景上，画面仍像 UI 控件；期望是：

- 默认状态是一片缓慢运行、呼吸感很弱的星云宇宙。
- 宇宙里有很多知识星球，写得越多的星球越亮、越大。
- 项目栏找不到自己的星球时，自动锁定当前项目的主领域星球。
- 点击星球先拉近并锁定视角，再次点击进入这个星球内部。

## 二、实际改动

### `panel.js`

- `universeDefsHtml()`：重写宇宙用渐变和滤镜，新增螺旋星云臂渐变 `galaxyArmGold` / `galaxyArmBlue`，降低普通背景色块的 UI 感。
- 新增 `galaxyArmPath()` / `renderGalaxyArms()`：用路径生成螺旋星云臂，不再只靠几块椭圆模拟星云。
- `renderStars()` / `renderGalaxyDust()`：星场从约 340 点提升到 620 点，星云尘埃从约 240 点提升到 560 点，默认画面更接近“星系”。
- `universeBallVisual()`：把默认知识星球压小，改成星云里的发光星体；仍保留“作品越多越大越亮”的比例规则。
- `renderUniverseBalls()`：新增星体外层 atmosphere，降低按钮感；锁定星球时放大到主体。
- `renderUniverseView()`：加入 `galaxyArmsLayer`，宇宙状态/图例改成隐藏，减少底部文字干扰。
- `applyUniverseCameraLock()`：锁定镜头从 `2.35` 提升到 `3.2`，让点击星球后更像被拉近星云。

### `panel.html`

- 宇宙区域改为更暗的全幅星云面板，弱化边框和工具栏。
- 动画速度进一步放慢：
  - `universeBreath`：210s
  - `galaxySlowTurn`：1320s
  - locked 状态：280s
- 工具栏、提示条、全局节点按钮默认低透明度，hover / focus 时再显现。
- 新增 `.galaxy-arm*`、`.universe-ball-atmosphere` 等样式。
- `.universe-status` / `.universe-legend` 在宇宙模式下隐藏。

### `tests/universe_fixture.html`

- 同步宇宙视觉 CSS，作为截图验收页。

### `tests/panel_map.test.js`

- 更新宇宙测试：
  - 要求星场不少于 560 个点。
  - 要求渲染至少 4 条螺旋星云臂。
  - 要求星尘不少于 520 个点。
  - 锁定镜头断言更新为 `matrix(3.2...)`。

## 三、验收标准与结果

| 验收项 | 结果 |
|---|---|
| 默认宇宙不再是金球 UI 堆叠 | 通过，默认星球压小为星云里的发光星体 |
| 星云有真实层次，而不是几块静态椭圆 | 通过，新增螺旋星云臂 + 密集星尘 |

| 再次点击进入单球 | 保持原逻辑，通过既有测试覆盖 |
| 作品越多，星球越亮越大 | 保持并通过测试 |

## 七、与预期相比

这次不是新增功能，而是修正视觉方向：从“带星云背景的金色 UI 球”调整为“星云里的知识星体”。默认画面更干净，锁定后有推进感，但仍然是 SVG/CSS 实现，不是 Three.js 或真实粒子引擎；后续如果还要继续拔高质感，可以单独做一次“地图视觉资产化/粒子化”任务。
