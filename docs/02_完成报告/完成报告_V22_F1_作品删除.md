# 完成报告：V22 F1 作品删除

时间：2026-05-09

## 本次改动

- `panel.js`
  - 资产作品详情页按钮从“删除这条记录”改为“删除作品”。
  - 新增 `deleteAssetRecord(rec, kind, btn)`，统一处理作品/速记删除。
  - 删除前二次确认文案改为：`确定删除这条作品？该文章和它的素材/立意会一并清除`。

  - 删除失败时用“球球删除失败：...”提示，并恢复按钮。

- `tests/panel_assets.test.js`
  - 新增作品详情删除测试：验证按钮文案、确认框文案、`dbDeleteArticle(id)` 调用、资产数量刷新、回到资产列表。

## 验收标准

- 资产作品详情页能看到红色危险态的“删除作品”按钮。
- 点击后出现二次确认，明确说明文章和素材/立意会一并清除。
- 确认后调用现有 `dbDeleteArticle(id)`。
- 删除后回到资产列表。
- `dbDeleteArticle(id)` 已在同事务中清理 `articles + nodes + clips`，本次不改 schema。

- 已符合：作品详情页删除入口和确认文案已改。
- 已符合：删除后走 `dbDeleteArticle(id)`，并刷新资产列表。
- 已符合：速记详情页也复用同一删除函数，文案保持为“删除速记”。
