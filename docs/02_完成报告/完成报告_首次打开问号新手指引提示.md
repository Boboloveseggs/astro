# 完成报告：首次打开问号新手指引提示

## 本次目标

确认第一次打开插件时，用户是否能看出来右上角的问号是「新手指引」入口；如果没有提示，就补一个轻量提示。

## 修改前状态

- 面板右上角已有 `?` 按钮。
- 该按钮只有 `title` / `aria-label` 提示，用户不悬停时看不出它是新手引导。
- 完整新手引导不会自动弹出，这一点符合「不要擅自替创作者决定」的要求。

## 本次改动

- 在 `panel.html` 的问号按钮旁边新增首次提示气泡。
- 提示文案为：
  - 标题：`新手指引在这里`
  - 正文：`想跟着教程走，就点问号；想自由探索，直接忽略也可以。`
  - 按钮：`知道了`、`开始指引`
- 在 `panel.js` 新增 `onboardingHintSeen` 记录：
  - 第一次打开且未完成新手引导时，只显示这个轻量提示。
  - 点击 `知道了` 后关闭提示，并记住不再自动提示。
  - 点击 `开始指引` 后关闭提示，并进入完整新手引导。
  - 点击右上角 `?` 也会关闭提示，再进入完整新手引导。
- 完整新手引导仍然不会第一次打开就自动弹出，只保留在问号入口里。

## 验收标准

- 第一次打开时，应能看到「新手指引在这里」的轻量提示。
- 提示不能遮挡主要操作区域，不能和宇宙 / 复盘 / 资产内容大面积重叠。
- 用户可以选择忽略，也可以主动开始教程。
- 完整教程不能自动强制出现。
- 不改 IndexedDB schema。
- 不改 Markdown 写入格式。
- 不改地图、宇宙、知识星球视觉参数。

## 实际验收

- 已通过截图验证：提示位于问号附近，没有强制打开完整引导。
- 已通过自动化测试：
  - `SUMMARY=126 passed / 0 failed`
- 已新增测试覆盖：
  - 第一次打开时只显示问号提示。
  - 不自动启动完整新手引导。
