# 完成报告：项目栏重新抓取按钮修复为原图标链路

完成时间：2026-05-10 18:35

## 问题

上一版项目栏小按钮会让 `background.js` 自己查询“当前标签页”。侧边栏场景下，这个判断可能拿不到真正的知乎页面，因此会显示无法抓取。

用户要求：这个按钮应当补回点击扩展图标的原能力，而不是另做一条不稳定的新链路。

## 修复内容

1. `panel.js`
   - 新增 `getCurrentZhihuTabForCapture()`。
   - 点击项目栏小按钮时，先在面板侧主动查找当前浏览器窗口里的知乎标签页。
   - 把明确的 `tabId/tabUrl` 发送给后台，而不是让后台猜。

2. `background.js`
   - 新增 `isZhihuUrl()` 和 `resolveCaptureTab()`。
   - 点击扩展图标时记录最近一次捕获的 tab。
   - 面板按钮传入 `tabId` 时，优先使用这个明确 tab。
   - 若没有传入 tab，再 fallback 到当前窗口 / 最近窗口 / 最近捕获 tab。
   - 仍然复用原来的 `captureFromTab()`，没有复制第二套抓取逻辑。

3. `tests/panel_analysis.test.js`
   - 更新项目栏按钮测试：确认按钮会把当前知乎页的 `tabId` 和 `tabUrl` 交给后台。
