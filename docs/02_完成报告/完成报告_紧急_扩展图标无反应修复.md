# 完成报告：紧急修复扩展图标无反应

完成时间：2026-05-09

## 问题

点击扩展图标后，可能出现 panel 不打开、知乎文章不抓取的问题。

审查结论：

- `background.js` 顶部同步 `importScripts('providers.js', 'analyzer.js', 'db.js')` 会让 service worker 变重。
- 一旦这些周复盘依赖加载失败或拖慢启动，核心监听 `chrome.action.onClicked` 就可能注册不上。
- 这会直接导致“点击图标无反应”。

## 修复原则

service worker 职责收窄为：

1. 点击图标。
2. 打开 side panel。
3. 抓取知乎当前页内容。

周复盘不再放在 service worker 中执行。

## 改动内容

### 1. background.js 最小化

文件：`background.js`

已删除：

- `importScripts('providers.js', 'analyzer.js', 'db.js')`
- 后台周复盘 alarm
- `generateWeeklyReportInBackground`
- `bgWeeklyStats / bgBuildWeeklyPrompt / bgNormalizeWeeklyReport` 等后台周复盘函数

保留并前置：

- `chrome.sidePanel.setPanelBehavior({ openPanelOnActionClick: false })`
- `chrome.action.onClicked.addListener(...)`
- `captureFromTab(tab.id)`
- `grabPageContent()`
- `chrome.runtime.onMessage` 对 `zhihu_article` 的兼容处理

### 2. 抓取函数恢复为自包含

文件：`background.js`

- `grabPageContent()` 不再引用 service worker 外层变量。
- 避免 `executeScript({ func })` 注入页面后因为外层变量不可见而失败。
- 保留知乎文章、回答、想法、feed 页候选块抓取。

### 3. 周复盘迁移到 panel.js

文件：`panel.js`

新增：

- `weeklyReportAutoPromise`
- `autoGenerateWeeklyReportOnPanelOpen()`

行为：

- 用户打开 panel 后，等待 storage 迁移完成。
- 如果本周报告已经存在，不重复生成。
- 如果本周有正式作品且配置了 API Key，就在 panel 中生成一次周复盘并保存。
- 如果用户稍后进入复盘页，会直接读取已生成报告。

## 手动复验步骤

1. 打开 `chrome://extensions/`。
2. 重新加载“知识图鉴”扩展。
3. 检查扩展卡片是否有红色错误按钮。
4. 打开知乎文章或回答页。
5. 点击扩展图标。
6. side panel 应立即打开。
7. panel 顶部应检测到当前知乎内容。

## 备注

这次修复的核心不是继续给 service worker 加兜底，而是把它减到最小。后续不要再把 AI 调用、IndexedDB 迁移、周报生成等重任务放回 `background.js`。
