# 完成报告：项目栏抓取按钮改为页面内抓取兜底

完成时间：2026-05-10 18:44

## 问题

项目栏新增的小按钮仍然提示没有抓到可分析内容。根因是：它虽然把 `tabId` 传给后台了，但后台只靠 `scripting.executeScript()` 注入抓取函数；在知乎页面结构变化、内容未完全渲染或侧边栏激活状态复杂时，容易拿不到正文。

## 修复内容

1. `background.js`
   - `captureFromTab()` 改为两段式抓取：
     - 第一优先：向知乎页里的 `content.js` 发送 `capture_request`，让页面内脚本自己读取正文；
     - 第二兜底：如果 content script 不响应或正文太短，再使用原来的 `chrome.scripting.executeScript()` 注入抓取。
   - 新增 `pickBestZhihuTab()`：当当前活动页不是知乎时，会从所有打开的知乎标签中选择最近访问的一个。
   - 注入抓取函数 `grabPageContent()` 增强：
     - 补充 `meta[property="og:title"]` 标题兜底；
     - 补充 `article/main/#root/body` 等正文兜底；
     - 避免页面结构小变动就直接抓取失败。

2. `panel.js`
   - `getCurrentZhihuTabForCapture()` 增强：
     - 先查当前窗口 / 最近窗口的活动知乎页；
     - 如果拿不到，再查所有已打开的知乎标签，选择最近访问的一个。
   - 新增 `pickBestZhihuCaptureTab()`。

3. `tests/panel_analysis.test.js`
   - 新增测试：当前活动页不是知乎时，按钮会回退到最近访问的知乎标签。
