# C1 审查问题修复完成报告

## 做了什么

### db.js
- 新增 `buildLocalMarkdownFilename(record)`：
  - 统一本地 Markdown 文件名生成逻辑。
  - 文件名仍保留标题可读性，但 shortId 稳定。
- `writeArticleToLocalFolder(record)` 写入前会清理同 shortId 的旧 `.md` 文件：
  - 如果用户在 C1 编辑器里改了标题，新文件写出前会删除旧标题文件。
  - 只删除当前绑定文件夹内、文件名以 `_${shortId}.md` 结尾的旧文件。
  - 跳过当前即将写入的新文件。
- `mergeParsedMarkdownRecord(existing, parsed)` 改为只保留真实变化：
  - 相同标题不再计为 patch。
  - 相同分析字段不再计为 patch。
  - 对对象字段做稳定排序比较，避免因为对象 key 顺序不同误判变化。
  - `nodes_hit` 内容没变时，不再触发节点表重建。

### panel.js
- `saveMarkdownEdit()` 遇到 `parseMarkdownRecord()` warning 时改为不保存：
  - 不写 IndexedDB。
  - 不写本地 Markdown。
  - 在编辑器里提示“发现格式问题，尚未保存”。
- 避免“用户写了异常节点行，保存后被旧规范内容覆盖”的体验问题。

### tests/db.test.js
- 新增测试：
  - `buildLocalMarkdownFilename: 标题参与文件名但 shortId 稳定`
  - `mergeParsedMarkdownRecord: 相同 Markdown 不产生 patch`
- 总测试数更新为 `37 passed / 0 failed`。

### 文档
- 更新 `完成报告_A4_最小单元测试.md`：`db.test.js` 从 23 个用例变为 25 个，总测试数变为 34。
- 更新相关完成报告和 `问题日志_Claude审核清单.md` 的最新测试计数。

| 步骤 | 预期结果 |
