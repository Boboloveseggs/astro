## 一、产品定位（最终版）

### 1.1 一句话定位

> **知识图鉴：让深度创作者看见自己的星空**

### 1.2 双轨同一产品

| 轨 | 用途 | 内容 |
|----|------|------|
| **自用版**（V2.1+V2.2）| 用户自己每天用 | 完整的创作流水线 + 自动归档 + 紫黑金宇宙 + OB 核心 |
| **比赛特化层**（不分叉）| 5/16 黑客松提交 | 在自用版基础上接直答 Agent + 刘看山（如官方提供素材）+ 演示叙事重写 |

**关键决策**：不做两个独立产品，而是**自用版作为底座，特化能力作为外接**。这是我之前推荐"双线并行"的修正。

### 1.3 主办方意图猜测（基于规则倒推）

| 推测意图 | 证据 | 权重 |
|---------|------|------|
| **拓宽板块** | 跟 Gemini 联动、给多类 API、不集中在改进现有业务 | 高 |
| **吸纳 AI 人才** | "AI 知乎实验室实习"承诺 | 高 |
| **营销话题度** | 让"知乎也搞 AI"被听到 | 高 |
| **改进社区** | 间接（次要）| 中 |
| **直接 ROI** | 不是主目的（赛事主要花钱）| 低 |

**结论**：主办方 50% 看技术能力 + 30% 看创意惊艳 + 20% 看生态贴合。**这意味着工程深度 + 独特视觉占优**。

## 三、V2.2 卖点扩展决策（已确认 G+C+A）

| 代号 | 卖点 | 工程量 | 价值 |
|------|------|-------|------|
| **G** | 写作风格指纹（基于 craft_review 历史归纳）| 0.5 天 | 让创作者看见"自己是谁"，情感价值极强 |
| **C** | AI 协同起草（用历史 clips + 立意 + 风格写新文章）| 2 天 | 创作者每天会用，强粘性 |
| **A** | 间隔重组推送（每周日推送复用建议）| 2 天 | "复利"概念是杀手锏 |

**已砍**（V2.x 不做，留 v2.2 之后）：
- 全平台抓取（Cubox 风格）：受众扩大 5x，但分散精力
- 月度成长报告（Day One 风格）：情感留存，但不紧急
- 学习者模式：用户群破圈，但偏离创作者核心
- 领域球分享链接：社交传播但有托管成本

## 五、获奖策略（已确认）

### 5.1 最终路径：自用版 + 轻量贴合

不做趋同的"直答 Agent 套壳"产品，而是**保留独特定位 + 轻量贴合**：

| 步骤 | 内容 | 工程量 |
|------|------|-------|
| 1 | V2.1 视觉打磨（用户和 GPT 直接沟通，Claude 不下指令）| 用户掌控 |
| 2 | V2.2 G/C/A 卖点扩展 | 5 天 |
| 3 | 追问对话切直答 Agent（5/12 API 放开后）| 1 天 |
| 4 | 刘看山 IP 接入（如官方发素材）| 0.5 天，可砍 |
| 5 | 100 次/天 调用量缓存 + 计数 UI | 0.5 天 |
| 6 | 演示叙事重写 + demo 录制 + zip 打包 | 2 天 |

合计 **9 天**（含 V2.1 视觉），距 5/16 还有 7 天，紧但可行。

### 5.2 直答 Agent 的演示卖点

> "原来的'语文老师标准'是我们 prompt 内置的（主观注入）。
> 现在用直答 Agent 评鉴，AI 基于**知乎海量优质回答的真实语料**判断创作质量——
> 这是**客观的、数据驱动的、有依据的**评鉴。"

**这是产品最强的差异化点**，必须在 demo 里突出讲。

### 5.3 不做的事

- ❌ 不做"直答助手"（趋同陷阱，60% 参赛者会做）
- ❌ 不做"刘看山陪聊"（情感产品偏离创作者定位）
- ❌ 不做"故事互动"（偏离自用版方向）
- ❌ 不双线并行（之前我推荐过，已修正：分散精力反而更糟）

### 5.4 获奖概率（最终估算）

| 路径 | 概率 |
|------|------|
| 自用版 + 卖点 + 接直答 + 演示讲好 | 30-45% 拿奖（含单项）|
| 进前三名 | 15-25% |
| 单项奖（最佳工程/最佳视觉/最佳创意/最佳贴合）| 30-50% |
| 完全陪跑 | 10-20% |

## 六、风险与不确定性（诚实记录）

### 6.1 主办方意图我们猜不准

我（Claude）和用户都不能 100% 确定知乎办这个比赛真正想要什么。这意味着所有"获奖策略"都是基于**猜测**的最优解。

### 6.2 评委构成不可知

- Gemini / 大厂 AI 人 → 看技术深度
- 知乎产品/运营 → 看生态贴合
- 不同评委权重导致不同获奖概率

### 6.3 趋同陷阱

100 个参赛者大概率有 60% 做"用直答 Agent + 刘看山套壳"。我们选择**不趋同 + 轻量贴合**，赌的是评委审美疲劳后会偏向独特作品。**这个赌可能赢，也可能输**。

### 6.4 GitHub 开源期待值

- 真实期望 1K-3K stars（前提：写好 README + 演示视频）
- 不是顶流但被尊重
- 不会被"AI 仔玩具"标签嘲讽（工程严谨度足够）
