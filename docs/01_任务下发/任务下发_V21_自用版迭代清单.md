## T2 球节点颜色调和（金色调统一）

**问题**：当前 `TYPE_META` 给四类节点分配了紫/绿/紫红/橙四种饱和色，与 1.0 深色金色主题严重冲突，"花花绿绿"。

**改动**：
1. **保留** `TYPE_META` 用于分析卡片中类型标签（资产详情、节点反链等），**类型语义不丢**
2. **新增** 一份地球专用色板 `EARTH_NODE_PALETTE`，球上的节点不再使用 `TYPE_META.color`
3. 球上节点统一用金色调家族：
   - person → 暖白金 `#e6c46a`
   - concept → 主金色 `#c8a44a`
   - theory → 深金 `#8a6c2e`
   - event → 橙金 `#d68b4a`
4. 节点的视觉差异**主要靠亮度等级**（半径 + alpha + glow）表达，颜色家族保持金色调
5. 节点 hover 卡中显示节点类型 label（用户仍能知道"这是人物/概念"）

**实现位置**：
- `panel.js` 第 1005 行 `earthTypeColor(type)` 改为查 `EARTH_NODE_PALETTE`
- `EARTH_NODE_PALETTE` 定义在 `panel.js` 顶部（紧邻 TYPE_META）

**验收**：
- 100 节点截图整体偏金色调，没有花花绿绿
- 不同类型节点仍有可分辨的色调差（暖白 vs 主金 vs 深金 vs 橙金）

## T3 球的实心立体感

**问题**：当前球只有经纬线骨架，缺乏球体感。

**改动**：在 `renderEarthSphere` 的 `<defs>` 加两个 radialGradient，球壳上额外渲染两个填充图层：

```html
<defs>
  <radialGradient id="earthSphereBody" cx="38%" cy="32%" r="68%">
    <stop offset="0%" stop-color="rgba(232,196,106,0.16)"/>
    <stop offset="55%" stop-color="rgba(140,110,60,0.10)"/>
    <stop offset="100%" stop-color="rgba(20,15,10,0.55)"/>
  </radialGradient>
  <radialGradient id="earthSphereGloss" cx="30%" cy="26%" r="20%">
    <stop offset="0%" stop-color="rgba(255,240,200,0.28)"/>
    <stop offset="100%" stop-color="rgba(255,240,200,0)"/>
  </radialGradient>
  <!-- 已有的 earthGoldGlow 保留 -->
</defs>

<!-- 在 sphereShell 之前插入 -->
<circle cx="cx" cy="cy" r="radius" fill="url(#earthSphereBody)"/>
<circle cx="cx" cy="cy" r="radius" fill="url(#earthSphereGloss)" pointer-events="none"/>
```

经纬线变弱：
- `.sphere-shell-line` 的 opacity 从 0.42 → 0.20
- `.sphere-shell-lat` 的 stroke-dasharray 保留，opacity 0.15

**验收**：
- 球看起来"实心"，中心暖、边缘暗
- 左上角有微弱高光（光源感）
- 经纬线变成辅助参考线（不喧宾夺主）
- 节点仍然清晰可辨（不被球面渐变盖住）
- 球面 SVG 仍可拖拽旋转（球面渐变是静态的，只有节点旋转）

## T4 复盘 Tab 锚点导航（子集跳转）

**问题**：复盘 Tab 内容从上到下堆叠，需要手动滚动，没人会往下翻。

**改动**：在复盘 Tab 顶部加一行 sticky 锚点条，点击跳到对应区块。

```html
<nav class="review-anchors">
  <button data-anchor="overview">本周概览</button>
  <button data-anchor="direction">创作方向</button>
  <button data-anchor="nodes">高频节点</button>
  <button data-anchor="timeline">时间轴</button>
  <button data-anchor="chat">AI 复盘</button>
</nav>

<section id="review-overview">...</section>
<section id="review-direction">...</section>
<section id="review-nodes">...</section>
<section id="review-timeline">...</section>
<section id="review-chat">...</section>
```

行为：
- 锚点条 `position: sticky; top: 0;` 始终在顶部
- 点击按钮 `scrollIntoView({ behavior: 'smooth', block: 'start' })`
- 滚动时根据当前可视区域**高亮当前锚点**（IntersectionObserver）
- 锚点样式：金色文字 + active 时金色下划线（与 Tab 风格一致）

**实现位置**：`panel.js` 的 `loadReview()` 内组装 anchors 行 + section id；新增 `bindReviewAnchors()`。

**验收**：
- 复盘 Tab 顶部出现 5 个锚点
- 点击锚点平滑滚动到对应区块
- 滚动时当前可视区块的锚点自动高亮
- 不影响数据卡 / 创作方向 / 时间轴 / 对话原本功能

## T7 OBSIDIAN 双链增强（地图内可见）

**问题**：用户希望 OBSIDIAN 双链能力直接在地图里露出，不只是浮按钮入口。

**改动**：
1. **Wikilink 虚拟边**：解析每篇文章的 `analysis.wikilinks`，如果文章 A 提到节点 B，则在该文章的所有 `nodes_hit` 节点和节点 B 之间画一条**虚线**（与 nodes_hit 共现的实线区分开）。
2. **节点 hover 卡新增"跨项目反链"分区**：当 hover 的节点在其他项目里也有同名节点时，分区显示"在【项目X】的【N 篇】作品里出现"。
3. **同名节点跨项目连线（保留到 T8 宇宙模式实现）**：本任务不做这条；宇宙视图下用银色细线连接。

**实现位置**：
- `panel.js` 的 `prepareMapData()` 增加 wikilink 边收集
- `renderEarthSphere()` 中链接渲染按 `link.kind === 'wikilink'` 用虚线
- `renderNodeHoverContent()` 增加"跨项目"分区

**验收**：
- 一篇文章里同时有 nodes_hit 节点和 wikilinks 节点 → 球上能看到实线（共现）和虚线（wikilink）
- 节点 hover 卡如果该节点跨项目存在，显示"在其他 N 个项目"提示
- 跨项目计数正确

## T8 宇宙模式（C 方案：混合）

**这是 V2.1 最大改动**。

### 8.1 数据层

新增 `prepareUniverseBalls(articles, mode)`：
- 输入：所有作品（`dbGetAllArticles()` 不带 project filter）+ 模式 `'project'` 或 `'domain'`
- 输出：球数组 `[{ ballId, name, articleCount, nodeCount, nodes }]`
- `mode === 'project'`：按 `rec.project_id` 分组，球名 = 项目名（从 `dbGetProjects()` 查）
- `mode === 'domain'`：按 `rec.analysis.domain` 分组，球名 = domain（空 domain 归到"未分类"球）
- 每个球的 `nodes` 用现有 `prepareMapData()` 逻辑算

### 8.2 视图状态

```js
let mapView = 'single';        // 'single' | 'universe'
let universeMode = 'project';  // 'project' | 'domain'
let activeBallId = activeProject;  // 当前进入的球（默认是当前项目）
```

切换规则：
- `mapView === 'single'`：渲染 3D 地球（已有 renderEarthSphere），节点是 `activeBallId` 这个球的节点
- `mapView === 'universe'`：渲染宇宙视图（多球阵列）
- 顶部地图工具栏新增按钮："**进入宇宙**"（在 single 模式时）/"**返回当前球**"（在 universe 模式时）
- universe 模式时再加一组按钮："按项目 / 按领域"切换 universeMode

### 8.3 宇宙视图渲染

新增 `renderUniverseView(balls, container)`：
- 用 SVG 在与单球相同的画布上渲染**多个小球**
- 球的中心位置 = Fibonacci 大尺度分布在画布上
- 球的半径 = 根据 `articleCount` 缩放（min=20px, max=60px）
- 球的样式 = 复用 T3 的 radialGradient（每个球都有立体感，但不画经纬线）
- 球面**不显示节点**（只显示球名 + 篇数标签）
- 鼠标 hover 球 → 球放大 + 显示完整信息（球名、N 篇作品、N 个节点、最常见 domain/项目）
- 鼠标 click 球 → `activeBallId = ball.ballId; mapView = 'single';` → 切回单球视图，显示该球的节点

### 8.4 宇宙→单球切换动画（简化）

不做完整的 zoom in 动画（避免工程复杂度）：
- 切换时容器淡出 200ms → 重新渲染 → 淡入 200ms
- 用户感知：响应迅速，过渡平滑

### 8.5 单球→宇宙切换

- 点工具栏"进入宇宙" → 当前球淡出 → 渲染 universe → 淡入
- 当前球的位置在宇宙阵列中**高亮**（金色边框 + 持续轻微脉动）

### 8.6 universeMode 切换

- 点"按项目 / 按领域"按钮 → universeMode 切换 → 重新调 prepareUniverseBalls(articles, mode) → 重渲染宇宙
- 切换时也用 200ms 淡入淡出

### 8.7 跨项目同名节点连线（universe 视角下）

在 universe 视图下：
- 找出在多个球里都出现的节点名（同名节点）
- 在两个球之间画一条银色细线（半透明 #aaa rgba(170,170,170,0.3)），表示有跨球关联
- 这是 OBSIDIAN 双链在宇宙视角下的视觉表达

### 8.8 进度记忆

- `mapView` / `universeMode` 用 `chrome.storage.local` 持久化
- 用户下次打开 panel 时恢复上次状态
