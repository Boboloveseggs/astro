## 0. 关键约定

### 0.1 插件小名：**球球**

GPT 在所有 UI 文案、加载动画、错误提示、报告语气里都要把插件称为"**球球**"。

例：
- 加载："球球正在思考…"
- 错误："球球找不到这篇文章了"
- 空状态："球球还没收到你的任何作品"
- 周复盘："球球替你看了这周写的 N 篇文章…"
- 月报："球球替你看了 N 月的所有作品…"

宇宙颜色 / 旋转 / 螺旋臂参数已经稳定。**所有调宇宙的指令都拒绝**，专注做这份清单。

### F7. 写作技法语气调整 — 0.3 天

**用户原话**："写作技法尽量模仿高中语文老师的夸奖，而不是通过结构用词节奏雅俗总评这种的分别列上。总评可以单独拿出来，写作技法这里主要是夸他，不要这么正式，显得很 AI"

**当前 prompt（错的）**：


**改后 prompt（对的）**：
```json
"craft_review": {
  "praise": "你的高中语文老师式夸奖：80-120 字。要像老师当着全班念你作文那样的语气——具体指出某段、某句、某个用词的好处，让创作者听到能笑出来。允许带轻微感叹（'真是…'、'最难得的是…'、'一般人写到这里都会…，你居然…'）。不要打分，不要罗列结构、用词、节奏等抽象维度——直接夸具体的句子或段落。",

```

**前端渲染调整**：
- 取消"结构/用词/节奏/雅俗"四块小卡
- 改成两块：
  1. **「语文老师夸你」卡** — 大字渲染 `praise`，下方红色波浪线高亮 `praise_quote`
  2. **「整体气质」卡** — 一句话总评（`summary`）
- 整体设计像作文本上老师的红笔批语风格

**parseAndValidate 兼容**：
- 旧记录有 `structure/diction/rhythm/elegance/summary`，新记录有 `praise/praise_quote/summary`
- 渲染时：优先用新字段，旧字段（structure 等）作 fallback 拼成一段简版

### G. 写作风格指纹 — 0.5 天 ⭐ 最划算

**做什么**：基于历史所有 `craft_review` 让 AI 归纳一段"你正在形成的写作风格"。

**位置**：复盘 Tab 顶部第一张卡（在周复盘之上 / 也可以是周复盘的副卡）。

**实现**：
1. 收集近 10 篇作品的 `craft_review.praise` 拼成一段
2. 调 LLM：
   ```
   prompt: '基于下面这些对同一创作者历篇文章的写作技法夸奖，归纳这位创作者正在形成的写作风格。要求：100-150 字，第二人称对话语气（"你"），具体到句法/材料选择/论证习惯/语言美学，不要贴标签，不要"专业的""有思想的"这种空话。'
   ```
3. 缓存：`chrome.storage.local.styleFingerprint = { generatedAt, content }`
4. 每月自动更新一次，或用户手动点"刷新指纹"

**渲染**：
- 卡片标题"你的写作指纹"
- 球球语气开头："**球球读了你的 N 篇文章后，发现你的写作有一个特别的味道**…"
- 大字段落（Noto Serif SC 14px）

**价值**：让创作者看见**自己是谁**——情感价值极强。

### C. AI 协同起草 — 2 天

**做什么**：用户选「素材 + 立意 + 领域 + 风格」让 AI 帮起草新文章。

**入口**：地图 Tab 右下角加按钮 `[用我的素材写一篇]`（紧邻"全局节点"按钮）。

**流程**：
1. **第一步：选取材**
   - 弹出半屏面板
   - 三个选区：
     - 素材库（多选 3-5 条）
     - 立意库（多选 1-2 条）
     - 领域（单选，从 dbGetProjects 取）
   - 写作风格自动套用最新的"风格指纹"（无需选）
   - 主题输入框（必填，30 字内说明这次想写什么）

2. **第二步：调 LLM**
   ```
   system: '你是一位深度创作者的写作伙伴，要用这位创作者自己积累的素材和立意写一篇新文章。要保留创作者已有的写作风格指纹。'
   user: `主题：{主题}
   领域：{领域}
   写作风格指纹：{styleFingerprint}
   可用素材：
   1. {素材1.content}（来自《{源文章1}》）
   2. {素材2.content}（来自《{源文章2}》）
   ...
   核心立意：
   - {立意1.viewpoint}
   - {立意2.viewpoint}
   
   要求：写一篇 800-1200 字的草稿。结构清晰，要把至少 2 条素材自然嵌入文章。开头有钩子，结尾有共鸣。`
   ```

3. **第三步：渲染草稿**
   - 弹出全屏抽屉
   - 标题区 + 正文区（Markdown 渲染）
   - 用了哪些素材会用金色高亮
   - 底部按钮：`[复制全文]` `[再生成一次]` `[保存为草稿到资产库]`

4. **第四步：保存为草稿**
   - 入库为 `article.type = 'draft'` 类型的文章
   - 资产 Tab 加"草稿"sub-tab，与"作品"并列

**价值**：创作者每天会**真正打开用**的能力——这是 V2.3 的最强粘性。

### A. 间隔重组推送 — 2 天

**做什么**：每周日推送一条"3 个月前的素材 + 这周的立意 = 新文章选题"建议。

**实现**：
1. **chrome.alarms 周期任务**：每周日早上 8:00 触发
2. **触发后逻辑**：
   - 取近 7 天的所有 `essence_insights`（这周的立意）
   - 取 60-180 天前的所有 `reusable_clips`（3-6 个月前的素材）
   - 调 LLM 配对：
     ```
     prompt: '下面是创作者最近 7 天提取的立意，和 3-6 个月前积累的素材。请找出 1-3 组「立意 × 素材」最有创作潜力的组合，每组写一段 50 字内的文章建议。'
     ```
3. **chrome.notifications**：发出系统通知
   ```js
   chrome.notifications.create({
     type: 'basic',
     iconUrl: 'icons/icon128.png',
     title: '球球的周日复盘建议',
     message: '你 3 个月前写过【损失厌恶】的素材，配上这周的【职场焦虑】立意，能写一篇新文章。',
     buttons: [{ title: '直接起草' }]
   });
   ```
4. 点击通知"直接起草"按钮 → 跳转到 C 卖点的"AI 协同起草"界面，自动填好对应的素材和立意

5. **历史推送**：所有推送过的内容保存在 `chrome.storage.local.recombinationLog`，复盘 Tab 第三张卡显示"球球给你的写作建议"列表

**价值**：让旧资产**自动变成新创作的肥料**——"复利"概念是杀手锏。
