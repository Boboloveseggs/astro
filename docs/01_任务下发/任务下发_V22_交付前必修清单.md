## 3. 各任务详细规格

### F1：资产作品删除

**问题**：当前资产 Tab 作品没有删除入口。

**改动**：
- 资产作品详情页加 `[删除作品]` 按钮（红色危险态）
- 点击弹出二次确认 `"确定删除这条作品？该文章和它的素材/立意会一并清除"`
- 调用现有 `dbDeleteArticle(id)`（已经会同事务清理 articles + nodes + clips）
- 删除后回到资产列表

**验收**：删除一条作品后，资产列表减少；地图节点反链该文章消失；素材库该文章 clips 消失。

---

### F2：顶部状态条扩充

**问题**：当前只有"总作品/总字数/本周进度"。

**改动**：状态条改为 4 列指标 + 1 行进度条：

```html
<div class="top-metrics">
  <div class="metric"><span class="metric-label">总</span><span class="metric-val">N 篇 · N 字</span></div>
  <div class="metric"><span class="metric-label">年</span><span class="metric-val">N 篇 · N 字</span></div>
  <div class="metric"><span class="metric-label">月</span><span class="metric-val">N 篇 · N 字</span></div>
  <div class="metric"><span class="metric-label">周</span><span class="metric-val">N 篇 · N 字</span></div>
  <div class="metric"><span class="metric-label">日</span><span class="metric-val">N 篇 · N 字</span></div>
  <div class="prog-wrap">…本周目标进度…</div>
</div>
```

数据来源：`dbGetAllArticles()` 后按 savedAt 切片。

布局：紧凑 5 列，进度条占满底部一行。

---

### F3：本周进度可设目标

**问题**：用户原话"那个本周 100% 是什么意思？我不太理解这个进度的概念"——目前默认是写满 5 篇 = 100%，但用户根本不知道。

**改动**：
- 设置页加"周创作目标"输入框（默认 3 篇）
- `chrome.storage.local.weekGoal` 持久化
- 顶部进度条文案改为：`"本周 N/M 篇"`（M 是目标，N 是已写）
- 进度条满 100% 时金色 + 文字"本周目标达成 ✓"

---

### F4：周复盘汇总报告（1.0 特色，重点）

**这是 Claude 之前漏掉的关键功能**。

**改动**：
1. **复盘 Tab 最顶部**新增 `[本周复盘报告]` 卡片（在锚点导航之上）
2. AI 生成本周复盘：调直答 Agent（5/12 后）/ 当前 LLM
   - 输入：本周所有作品的 title + core_claim + creator_strength + craft_review
   - 输出（结构化）：
     - **本周亮点**：你这周写得最好的 1-2 篇
     - **创作模式**：你这周形成的写作风格特征
     - **空白机会**：根据本周作品 + 历史地图，下周建议方向
     - **数据**：本周 N 篇 / N 字 / 涉及 N 个领域
3. 生成时机：每周日晚 8 点自动跑一次（chrome.alarms），用户手动点"刷新"也行
4. 缓存到 `chrome.storage.local.weeklyReports[YYYY-WW]`，每个周一份
5. 历史周报可往下翻（最近 4 周）
6. 文案语气：**球球的口吻**，不是冷冰冰的 AI 总结
   - 例如开头："**球球看了你这周写的 N 篇文章，最让它眼前一亮的是…**"

**1.0 引用**：参考 1.0 demo 脚本里"周复盘"概念。

### F5：hover 卡变慢回归

**问题**：用户反馈"鼠标放节点上不显示关联文章了，或显示很慢"。

**排查方向**：
1. 17:38-17:58 三次旋转修正可能引入了 hover 延迟（pointermove 节流过度？）
2. NODE_HOVER_SHOW_DELAY = 1000 是否被改了？
3. requestAnimationFrame 跟自转互相抢？

**改动**：
- 验证 `NODE_HOVER_SHOW_DELAY` 仍为 1000ms
- 如果 GPT 在拖拽期间禁用了 hover，要改为"拖拽结束 200ms 后恢复 hover"
- 测试用例新增：连续旋转后停下，hover 节点应在 1 秒内浮出卡片

### F7：写作技法语气调整

**问题**：用户原话"主要是夸他，不要这么正式，显得很 AI"。

**当前 prompt**：
```
"craft_review": {
  "structure": "起承转合 / 论证层次评价（一句）",
  "diction": "用词精度评价（一句）",
  "rhythm": "长短句节奏 / 段落呼吸（一句）",
  "elegance": "雅俗度 / 是否易读（一句）",
  "summary": "用初高中语文老师视角总评一句，鼓励为主，问题点到为止，不打分"
}
```

**改后 prompt**：
```
"craft_review": {
  "summary": "你的高中语文老师式夸奖：80-120 字。要像老师当着全班念你作文那样的语气——具体指出某段、某句、某个用词的好处，让创作者听到能笑出来。允许带轻微感叹（'真是…'、'最难得的是…'、'一般人写到这里都会…，你居然…'）。不要打分，不要罗列结构、用词、节奏等抽象维度——直接夸具体的句子或段落。",
  "praise_quote": "从原文里摘一句你最想夸的话（≤30 字）"
}
```

**前端渲染**：
- 取消"结构/用词/节奏/雅俗"四块小卡
- 只显示一段大字"语文老师夸你"+ 红笔下划线高亮 `praise_quote`
- 整体设计像作文本上老师的红笔批语
