# 紧急修复：点击扩展图标无反应

> 问题报告：用户在知乎页面点击扩展图标，没有任何反应（panel 不打开 / 不抓文章）。
> 定位：`background.js` 第 3-7 行新加的 `importScripts('providers.js', 'analyzer.js', 'db.js')` 极有可能让 service worker 崩溃。
> 后果：`chrome.action.onClicked` 监听根本没注册，所以点击图标无反应。

## 1. 根因分析

### 1.1 importScripts 是同步且不能完全被 try-catch 兜底
- 如果加载的脚本有 SyntaxError → service worker 整体崩溃
- 即使没有 SyntaxError，加载 1000+ 行的 db.js（含 IndexedDB 逻辑）也会让 worker 启动慢
- worker 启动失败 → action.onClicked 监听不上 → 点击图标无反应

### 1.2 关键风险点
`background.js` 当前结构：
```js
try {
  importScripts('providers.js', 'analyzer.js', 'db.js');  // ← 危险
} catch (e) {
  console.warn(...);
}

// ↓↓↓ 后面的代码依赖上面 importScripts 成功
chrome.runtime.onInstalled.addListener(...)
chrome.action.onClicked.addListener(...)  // ← 如果 importScripts 抛 SyntaxError，这里就注册不上
```

## 2. 修复方案（按优先级）

### 方案 A（首选，零风险）：把 onClicked 监听移到最前面

**核心思想**：先注册关键监听器，再加载依赖。这样即使 importScripts 失败，**点击图标依然能开 panel + 抓文章**（核心功能不受影响）。

```js
'use strict';

// ========== ① 第一优先：核心功能监听器（不依赖周复盘）==========
chrome.sidePanel.setPanelBehavior({ openPanelOnActionClick: false }).catch(() => {});

chrome.action.onClicked.addListener((tab) => {
  // 1. 同步打开 panel（必须在用户手势内）
  chrome.sidePanel.open({ tabId: tab.id }).catch((e) =>
    console.warn('[知识图鉴 bg] 开面板失败:', e.message)
  );

  // 2. 异步抓取知乎内容
  if (!tab.url?.includes('zhihu.com')) return;
  captureFromTab(tab.id);
});

chrome.runtime.onMessage.addListener((msg, sender, sendResponse) => {
  if (msg.action === 'zhihu_article') {
    chrome.storage.local.set({ lastArticle: msg.article });
    sendResponse({ ok: true });
  }
});

chrome.runtime.onInstalled.addListener(() => {
  chrome.sidePanel.setPanelBehavior({ openPanelOnActionClick: false }).catch(() => {});
});

// captureFromTab + grabPageContent 函数定义（保留原样）
async function captureFromTab(tabId) { /* ... */ }
function grabPageContent() { /* ... */ }

// ========== ② 第二优先：周复盘后台生成（可降级）==========
let weeklyReportReady = false;
try {
  importScripts('providers.js', 'analyzer.js', 'db.js');
  weeklyReportReady = true;
} catch (e) {
  console.warn('[知识图鉴 bg] 周复盘依赖加载失败，主功能仍可用:', e.message);
}

if (weeklyReportReady) {
  // 周复盘相关代码（alarm 注册 / generateWeeklyReportInBackground 等）
  const WEEKLY_REPORT_ALARM = 'zhijing_weekly_report';
  // ... 原有的 bgStartOfWeek / bgWeeklyStats / bgBuildWeeklyPrompt / 等等所有周复盘函数
  // ... scheduleWeeklyReportAlarm + chrome.runtime.onStartup + chrome.alarms.onAlarm
}
```

**关键改动**：
1. **chrome.action.onClicked 监听**移到文件最前面
2. importScripts 后置且失败不影响主功能
3. 周复盘相关代码全部包在 `if (weeklyReportReady)` 内

### 方案 B（备选）：周复盘改用 chrome.storage 临时数据，主分析在 panel 里跑

不在 service worker 里调 LLM，让 panel 打开时自己检查"本周是否已生成周报"，如未生成且条件满足就在 panel 里跑。

这样 background.js 完全不需要 importScripts。

**采用方案 A 即可**，方案 B 只在 A 不奏效时备用。

## 3. 验收

### 3.1 必须自测的步骤

GPT 改完后必须：
1. 在 chrome://extensions/ 重新加载扩展
2. **看扩展卡片有没有红色"错误"按钮** —— 如果有，点进去看 service worker 报错
3. 在 service worker 详情页（chrome://extensions/ 点扩展卡片的"service worker"）打开 console，看是否有红色错误
4. 在知乎打开任意一篇文章
5. 点击扩展图标
6. **side panel 必须打开**（这是 #1 优先）
7. **panel 顶部 detected card 显示该文章**（说明抓取成功）

### 3.2 如果方案 A 仍不奏效

- 周复盘报告改成 panel 打开时检查 + 生成（panel.js 跑）
- service worker 只保留点击图标 + 抓取逻辑
