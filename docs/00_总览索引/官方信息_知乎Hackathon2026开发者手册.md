# 官方信息：知乎 Hackathon 2026｜AI 脑洞实验室开发者手册

更新时间：2026-05-11

## 一、官方背景摘录

过去 15 年，知乎一直致力于把人与知识、人与人连接到一起。从一个好问题，到一篇硬核回答，再到一段真诚分享，知乎构筑起中文互联网中具有厚度的知识思想连接与人文关系网络。

2026 年，技术革新正在重构连接的底层逻辑。AI 不再只是效率工具，它正在成为思维外延、表达助力与数字分身，让知识与人的连接更加鲜活、高效。

官方提出的核心问题：

- 如果 AI 成为人与人、人与内容之间的全新媒介，知乎体验会被如何重新设计？
- 如果知乎上数以亿计的真实讨论遇上前沿 AI 技术，会产生怎样的全新可能？
- 能不能跟 AI 一起，在知乎做出前所未有的新东西？

官方活动表达：

> 知乎 Hackathon｜AI 脑洞实验室不谈宏大空洞的技术概念，而是鼓励参赛者把灵感落地，让脑洞成真。

## 二、开放能力与限制摘录

### 1. 全网搜索 / 内容检索相关能力

官方强调可以利用作者、发布时间、相关性分数、权威度等级等信号筛选优质内容，提升作品专业度。

限制：

- 单用户总调用量上限：`1000 次 / 天`
- 必须在应用层做好缓存
- 避免频繁实时搜索

### 2. 知乎知识

官方方向：

- 用创意把枯燥知识内容以更易理解的形式呈现给观众
- 搭建讨论模块，让观众一同讨论热点知识
- 鼓励围绕知识内容做新的呈现方式

### 3. 直答 Agent

能力说明：

- 接口：`POST /v1/chat/completions`
- 用于生成专业、可信、精炼的自然语言回答
- 可以基于知乎优质内容辅助回答问题

可做方向：

- 根据用户问题生成有据可依的回答
- 搭建问答工具、内容助手、智能咨询模块
- 结合搜索与关注流信息，让输出贴合用户偏好与社区语境

限制：

- 单用户总调用量上限：`100 次 / 天`
- 必须在应用层缓存
- 避免频繁实时请求

## 三、社区开放 API 关键信息

Base URL：

```text
https://openapi.zhihu.com/
```

鉴权方式：

- `app_key`：用户 token
- `app_secret`：知乎提供的应用密钥
- 使用 HMAC-SHA256 签名
- 请求头包含：
  - `X-App-Key`
  - `X-Timestamp`
  - `X-Log-Id`
  - `X-Sign`
  - `X-Extra-Info`

当前白名单圈子：

| 圈子 ID | 圈子名称 |
| --- | --- |
| `2001009660925334090` | OpenClaw 人类观察员 |
| `2015023739549529606` | A2A for Reconnect |
| `2029619126742656657` | 黑客松脑洞补给站 |

社区接口限制：

- 全局限流：`10 QPS`
- 发布想法：每小时最多 `5` 条
- 单个想法下创建评论：每小时最多 `20` 条
- 禁止批量、高频、无意义调用
- 禁止刷屏、恶意灌水、重复投稿、垃圾内容批量推送

## 四、黑客松故事开放内容库

官方为 2026 黑客松特殊开放会员小说内容库。

### 获取故事列表

```text
GET /openapi/hackathon_story/list
```

返回字段包括：

- `work_id`
- `title`
- `artwork`
- `tab_artwork`
- `description`
- `labels`

### 获取故事详情

```text
GET /openapi/hackathon_story/detail?work_id=xxx
```

返回字段包括：

- `work_id`
- `chapter_name`
- `author_avatar`
- `author_name`
- `labels`
- `introduction`
- `content`

正文限制：

- 最多返回 `3000` 字
- 仅用于本次活动
- 传播时需要署名作品及作者

## 五、与知识图鉴当前比赛版的对应关系

当前比赛版已接入：

| 官方能力 | 当前接入方式 | 状态 |
| --- | --- | --- |
| 黑客松故事列表 | `GET /openapi/hackathon_story/list` | 已接入 |
| 黑客松故事详情 | `GET /openapi/hackathon_story/detail` | 已接入 |
| AI 分析能力 | 使用比赛版内置智谱 Key 兜底 | 已接入 |
| 知识资产整理 | 故事 / 文章进入资产、素材、立意、知识地图 | 已接入 |

当前暂不接入：

| 官方能力 | 暂不接入原因 |
| --- | --- |
| 发布想法 | 社区写入能力，有频率与违规风险；当前产品主线不是自动发帖 |
| 创建评论 / 删除评论 | 社区写入能力，容易偏离创作者知识整理场景 |
| 点赞 / 取消点赞 | 社区互动能力，不是当前核心体验 |
| OAuth 登录 | 浏览器插件核心体验不依赖用户登录，短期接入会增加回调与 token 管理复杂度 |
| 直答 Agent | 可作为后续增强项；当前比赛包已用智谱分析兜底，避免在提交前重构 AI 层 |

## 六、当前比赛版叙事口径

知识图鉴 Astro 不是一个自动发帖或互动机器人，而是面向知乎创作者的个人知识整理插件。

它把用户主动打开或选择的知乎内容、黑客松开放故事内容，转化为：

- 文章结构复盘
- 可复用素材
- 立意资产
- 知识节点
- 知识宇宙
- 周复盘报告

与官方主题的契合点：

- 用 AI 重新设计“人与内容”的连接方式
- 让知乎内容不只是被读过，而是沉淀为创作者自己的知识资产
- 通过知识宇宙和周复盘，把零散内容变成长期创作地图
- 使用官方开放故事内容库，展示知乎开放内容如何进入创作者工作流

## 七、合规说明

当前比赛版坚持：

- 用户主动点击触发
- 不批量调用
- 不高频调用
- 不自动发布社区内容
- 不自动评论
- 不自动点赞
- 不获取他人隐私
- 不绕过页面权限
- 对开放 API 调用保持克制，并优先选择只读内容接口
