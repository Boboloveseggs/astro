# 知乎黑客松 API 申请清单

时间：2026-05-11

## 一句话结论

知识图鉴当前最需要申请的不是全部 API，而是这三类：

1. **知乎 OAuth / 登录相关权限**
2. **直答 Agent**
3. **知乎搜索**

有余力再申请：

4. **知乎热榜**

暂时不建议作为首版核心：

- 知乎故事
- 知乎关注流
- 圈子发帖 / 评论 / 点赞
- 全网搜索

## 必须申请：OAuth / 登录权限

### 为什么必须

比赛提交页要求“作品链接可供社区用户点击登录体验”。如果你的线上 demo 或项目入口需要让评委以知乎账号进入，就必须配置：

- App_ID
- App_KEY
- 知乎登录回调地址

否则评委可能点开后无法授权登录，平台也可能无法统计产品登录数。

### 用在知识图鉴哪里

- 评委打开线上 demo
- 点击“用知乎登录体验”
- 登录后进入演示版知识图鉴
- 后续可把用户身份用于保存自己的作品图鉴、区分演示数据和个人数据

### 申请时怎么描述

> 用于知识图鉴作品的评委体验登录。用户通过知乎账号登录后，可以进入自己的创作者知识整理空间，查看文章分析、知识资产、知识地图和总复盘报告。

## 强烈建议申请：直答 Agent

接口能力：

`POST /v1/chat/completions`

### 为什么需要

你现在插件里的 AI 分析主要依赖用户自己填 API Key。比赛评委不一定愿意配置第三方 API Key，也不一定知道怎么配。

直答 Agent 可以作为比赛版默认 AI 能力，让评委不用填第三方 Key，也能看到：

- 文章核心观点分析
- 写作技法复盘
- 可复用素材提炼
- 下一篇选题建议
- 周复盘总结

### 用在知识图鉴哪里

优先接这几个点：

1. “开始分析”按钮
2. 单篇文章追问
3. 总复盘报告
4. 下周选题建议

### 注意限制

官方说明是单用户总调用量 100 次 / 天，所以必须做缓存：

- 同一篇文章不要重复分析
- 同一个周报不要反复生成
- 演示模式优先走内置数据，减少真实调用

### 申请时怎么描述

> 用于帮助知乎创作者对自己的公开文章进行结构复盘、素材提炼、写作技法分析和下一篇选题建议。系统会对相同文章结果做缓存，避免高频重复调用。

## 强烈建议申请：知乎搜索

接口能力：

`GET /api/v1/content/zhihu_search`

### 为什么需要

你的产品核心是“创作者整理自己的知乎作品”。如果只有插件读取当前页面，评委会觉得它像一个本地工具。接入知乎搜索后，它和知乎生态的关系会更明显：

- 可以按关键词搜索知乎内容
- 可以找到相关问题、文章、优质回答摘要
- 可以给创作者提供选题依据
- 可以让知识图鉴从“整理已写作品”扩展到“发现下一篇可写方向”

### 用在知识图鉴哪里

建议最小接法：

1. 在“总复盘报告”里增加“下周知乎选题灵感”
2. 在“下一步选题建议”旁边增加“查找知乎相关讨论”
3. 在资产页增加“按关键词发现知乎参考内容”

### 注意限制

官方说明是单用户总调用量 1000 次 / 天。

必须缓存：

- 同一个关键词 24 小时内复用结果
- 不要边输入边请求
- 用户点击按钮后再请求

### 申请时怎么描述

> 用于帮助创作者围绕自己的知识资产，在知乎站内检索相关高质量讨论、问题和文章，为下一篇创作提供选题参考。系统会按关键词缓存结果，避免频繁请求。

## 建议申请：知乎热榜

接口能力：

`GET /api/v1/content/hot_list`

### 为什么建议

热榜很适合做“选题雷达”，但不是知识图鉴的核心。它可以锦上添花，让作品更像知乎比赛作品。

### 用在知识图鉴哪里

1. 总复盘页：本周可借势热点
2. 资产页：把自己的高频主题和热榜话题做匹配
3. 下一篇建议：如果某个热点和自己的知识球相关，就提示可以写

### 注意限制

官方说明是单用户 100 次 / 天，必须缓存。

### 申请时怎么描述

> 用于帮助知乎创作者把自己的长期知识资产与当下知乎热点建立连接，发现可以继续创作的公共议题。系统会定时缓存热榜，不做高频刷新。

## 暂时不建议申请为核心：全网搜索

接口能力：

`GET api/v1/content/global_search`

### 原因

知识图鉴要强调知乎生态，首版用“知乎搜索”更贴题。全网搜索容易把作品拉偏成普通搜索工具。

### 什么时候再用

后续做“站外资料补充”时再接。

## 暂时不建议申请为核心：知乎故事

### 原因

知乎故事适合改编、续写、互动阅读。你的作品是创作者作品复盘和知识资产整理，主线不匹配。

### 风险

强行接故事接口，会让作品定位变散。

## 暂时不建议申请为核心：关注流

### 原因

关注流适合个性化内容推荐和社交关系分析，但你的当前产品还没有做“关注作者动态”这条线。

### 什么时候再用

后续如果做“关注的创作者给我带来哪些选题刺激”，可以接。

## 暂时不建议申请为核心：圈子发布 / 评论 / 点赞

### 原因

圈子接口偏社区互动、发帖、评论、点赞。当前知识图鉴是个人知识整理工具，不需要自动发布内容。

### 合规风险

发布、点赞、评论接口容易触碰官方强调的“禁止批量、高频、无意义调用”。比赛首版不建议碰。

## 最小可提交方案

如果时间很紧，只申请并接：

1. OAuth / 登录
2. 直答 Agent

这样评委至少可以：

- 打开作品链接
- 用知乎登录
- 进入知识图鉴
- 看到演示数据
- 点击分析 / 复盘时有知乎官方智能能力兜底

## 推荐完整申请方案

建议你去申请这四个：

1. OAuth / 登录权限
2. 直答 Agent
3. 知乎搜索
4. 知乎热榜

这四个和知识图鉴最匹配。

## 不要这样申请

不要说：

> 我要批量抓取知乎内容，批量分析用户主页文章。

要说：

> 用户登录后，系统帮助创作者整理自己已打开、已选择或主动输入的内容，并结合知乎站内搜索、热榜和直答 Agent，生成文章复盘、知识资产和下一步选题建议。系统会做缓存，不进行高频、批量、无意义调用。

## 给申请表的简短描述

> 知识图鉴 Astro 是一个面向知乎创作者的个人作品复盘与知识资产整理工具。用户可以把自己正在阅读或创作的知乎文章整理进个人知识库，系统会生成结构复盘、素材沉淀、知识地图和周复盘报告。我们希望申请 OAuth 登录、直答 Agent、知乎搜索和知乎热榜接口，用于评委登录体验、文章分析、站内选题参考和热点匹配。所有接口调用都会基于用户主动操作触发，并进行缓存，避免批量和高频请求。
